Overview

What you should know by the end of this lesson

Data processing questions usually start with raw input and end with decisions. You need to explain how data is captured, how the system transforms it, and why each step matters.

Learning goals

  • Explain how hardware and software collect data.
  • Define the major data processing functions clearly.
  • Link processing steps to useful outputs and decisions.
  • Apply data-processing ideas to real scenarios.

Quick memory hooks

  • Collect: gather the raw data.
  • Validate: check it follows rules.
  • Process: turn it into something useful.
  • Report: present the outcome clearly.

Where this appears

These ideas appear in tills, sensors, school registers, online forms, business dashboards, and any system that turns raw inputs into information people can act on.

Collection

Data can be collected by people, devices, or automated systems

Collection is the first stage. Hardware captures the raw data, while software controls forms, rules, timing, and how that data enters the system.

Hardware role Devices such as keyboards, scanners, cameras, sensors, and barcode readers capture raw input.
Software role Software prompts users, accepts values, applies rules, and sends data onward for processing.
Manual collection A user types or selects information, which can be flexible but may introduce mistakes.
Automatic collection Sensors or system logs can gather data continuously or on a schedule with little human input.

Example

In a school canteen, a barcode scanner collects the item code, the till software checks the product table, and the system then processes the sale into stock and revenue data.

Functions

Function 1

Validation checks data against rules

Validation happens during or just after collection. It checks whether data is sensible enough to process, such as a required field being filled in, a number being within range, or a date following the expected format.

Age: 16 range 0-120 accepted

Exam phrase: validation improves data quality, but it does not prove that the data is true.

Function 2

Sorting places records into a useful order

Sorting changes the order of records without changing their meaning. A system might sort students by surname, orders by date, products by price, or help desk tickets by priority.

Delta Alpha Charlie A-Z Alpha Charlie Delta

Exam phrase: sorting makes records easier to search, compare, prioritise, or present.

Function 3

Conversion changes data into a different form

Conversion is used when data has been collected in one format, unit, encoding, or type but another part of the system needs something different. It can convert units, file formats, data types, or coded values.

19.5 C convert 67.1 F

Exam phrase: conversion keeps data compatible with the next process, database, output, or user need.

Function 4

Aggregation combines records into summaries

Aggregation takes many individual values and produces a grouped result, such as a total, count, average, minimum, or maximum. This reduces detail so the overall picture is easier to see.

4 7 6 sum 17

Exam phrase: aggregation is useful when managers need totals or summaries rather than every raw record.

Function 5

Analysis looks for patterns and meaning

Analysis interprets processed data to find trends, exceptions, relationships, or likely causes. It helps a user move from knowing what happened to understanding why it may have happened.

Mon Tue Wed Thu rising trend

Exam phrase: analysis supports decisions because it explains patterns, unusual values, or changes over time.

Function 6

Reporting presents the result clearly

Reporting turns processed data into an output for a person or organisation. It might be a dashboard, chart, exception report, printed statement, alert, or management summary.

summary sent

Exam phrase: reporting matters because the information must be understandable, timely, and suited to the audience.

Output

Processed results are useful because they support action and decision-making

Raw data on its own often has limited value. Processing helps users understand what is happening and what to do next.

Input raw values Processing checks and transforms Output useful information

Worked judgement

A weather station may collect temperature readings every minute, validate that the values fall within realistic limits, sort the readings by time, analyse trends across the day, and report the results in a graph for staff to review.

Exam Traps

Common mistakes with data collection and processing

Most lost marks come from mixing up the stages or using the wrong technical word for the job the system is doing.

Confusing validation with verification

Validation checks rules, not whether the data is definitely true.

Fix: Describe validation as a rule check such as range, format, or presence.

Jumping straight to analysis

Many systems need sorting, conversion, or aggregation first.

Fix: Explain the sequence from input to processed output.

Ignoring hardware and software roles

Collection usually depends on both.

Fix: Say what device captures the data and what software manages it.

4 marks Explain

Question 1

Explain the difference between collecting data and processing data using a supermarket checkout example.

Show answer guide
  • Collection is the capture of raw data, such as scanning a barcode.
  • Processing is what the system does with it afterward.
  • The system may look up the product, price, or stock record.
  • It may then total the bill or update inventory.
6 marks Discuss

Question 2

A school wants to analyse attendance patterns. Discuss how data could be collected and processed to produce useful reports for staff.

Show answer guide
  • Attendance data could be collected through registers, cards, or a digital form.
  • Software could validate missing or invalid entries.
  • Records could be sorted by class, date, or student.
  • Aggregation could total absences or late arrivals.
  • Analysis could identify patterns across groups or time periods.
  • Reporting could present charts or summaries for staff action.

Conversion
