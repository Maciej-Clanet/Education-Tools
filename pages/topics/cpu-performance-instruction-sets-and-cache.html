Overview

What you should know by the end of this lesson

CPU performance is more than just clock speed. In Unit 2, you need to explain what affects execution speed, how instruction-set design changes CPU behaviour, and why cache helps reduce delays caused by slower main memory.

Learning goals

  • Describe factors that affect CPU execution speed.
  • Explain what an instruction set is and why it matters.
  • Explain how cache improves performance.
  • Discuss trade-offs such as cost, heat, and power use.

Quick memory hooks

  • Execution speed: how quickly the CPU completes work.
  • Instruction set: the collection of operations the CPU can understand.
  • Cache: small, fast memory close to the CPU.
  • Implication: more speed often means more cost, heat, or power use.

Where it appears

These ideas matter whenever you compare processor choices, explain why one system feels faster than another, or justify design decisions in exam scenarios.

Performance

Execution speed depends on several linked factors

Clock speed matters, but it is not the whole story. Real CPU performance is affected by how quickly instructions can be processed, how often the CPU waits for memory, and how efficient the overall design is for the task being run.

Clock speed more cycles each second Cache fewer waits for RAM Efficiency useful work done per cycle also matters
Clock speed A higher clock can let the CPU complete more cycles each second, but only within thermal and power limits.
Cache size and design Better cache reduces the need to wait for slower main memory when data is reused often.
Processor design An efficient microarchitecture can complete more useful work per cycle.
Implications Higher performance can increase cost, heat output, and power consumption.

Why a faster number is not always a faster CPU

Two processors with similar clock speeds can behave differently if one spends less time waiting for memory, has a better cache, or needs fewer cycles to complete the same task.

Instruction Sets

The instruction set affects what the CPU can do and how it does it

An instruction set is the collection of machine-level operations the processor understands. Choosing an instruction-set style has implications for compatibility, complexity, power use, and how efficiently software runs on the processor.

Choice Typical features Possible implication
Larger or more complex instruction set More built-in operations and sometimes more powerful single instructions. Can reduce program length, but may increase decoder complexity and power use.
Smaller or simpler instruction set Fewer instruction types with a more regular design. Can improve efficiency and energy use, but may need more instructions to complete a task.
Compatibility focus Supports a particular family of software and operating systems. Changing instruction-set family can affect whether existing software will run directly.

What to say in an exam answer

Do not just say that the instruction set is "instructions." Stronger answers explain that it defines the operations the CPU can understand and affects efficiency, compatibility, and design choices.

Cache

Cache helps the CPU avoid repeated delays when data is reused

Cache is a small amount of very fast memory located close to the CPU. It stores frequently used instructions or data so the processor does not always have to wait for slower RAM.

Cache hit

The needed instruction or data is already in cache, so the CPU can continue quickly.

Cache miss

The CPU must fetch from slower main memory, causing a delay before work can continue.

Why it works

Programs often reuse nearby instructions and data, so keeping popular items close to the CPU saves time.

Memory type Speed Typical role
Cache Very fast Holds frequently used instructions or data close to the CPU.
Main memory (RAM) Slower than cache Stores programs and data currently in use by the system.
Secondary storage Much slower Stores files and programs long term.

Exam Traps

Common mistakes students make with performance and cache

These topics often lose marks when answers become too vague or focus on one single factor without explaining the trade-offs.

Assuming clock speed is everything

A higher GHz figure does not automatically make a CPU better in every situation.

  • Cache design also affects performance.
  • Instruction efficiency and processor design matter too.

Fix: Mention at least one other factor besides clock speed.

Treating cache like permanent storage

Cache is not a place for saving files long term.

  • Cache is small, fast memory used during active processing.
  • It helps the CPU access reused data more quickly.

Fix: Link cache to speed and short-term access, not file storage.

Defining an instruction set too loosely

Saying "it is the instructions in a program" is not precise enough.

  • It is the set of operations the CPU can understand.
  • It affects compatibility and processor design choices.

Fix: Use the phrase operations the CPU can understand.

4 marks Explain

Question 1

Explain how cache improves CPU performance.

Show answer guide

A strong answer links cache to reduced waiting.

  • Cache is small, fast memory close to the CPU.
  • It stores frequently used instructions or data.
  • If the CPU finds the needed item in cache, it avoids a slower RAM access.
  • This reduces delays and improves overall execution speed.
6 marks Discuss

Question 2

A device maker is comparing two CPU designs: one aims for low power use with a simpler instruction-set approach, and the other aims for maximum speed with a larger cache and higher performance target. Discuss the advantages and implications of each choice.

Show answer guide

The best answers compare benefits and trade-offs on both sides.

  • A simpler, lower-power design may improve energy efficiency and battery life.
  • It may suit devices where heat and power limits matter more than raw speed.
  • A higher-performance design with more cache can reduce waiting for memory and improve speed.
  • That may be better for demanding workloads.
  • However, higher performance often increases cost, heat output, and power consumption.
  • The final choice depends on the purpose of the system being built.

Glossary

Quick glossary

Execution speed
How quickly a CPU can complete useful processing work.
Instruction set
The collection of operations the CPU can understand and execute.
Cache
Small, fast memory close to the CPU used for frequently needed data or instructions.
Cache miss
A situation where the needed item is not in cache, so slower memory must be used.

Remember

Performance is a trade-off topic

  • Clock speed matters, but it is not the whole story.
  • Cache cuts down waiting for slower memory.
  • More speed can mean more heat, cost, or power use.
