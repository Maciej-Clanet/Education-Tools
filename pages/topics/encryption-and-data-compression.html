Overview

What you should know by the end of this lesson

Transmission is not only about moving bits. The data may also need protection from unauthorised reading and may need to be made smaller so it can be stored or sent more efficiently.

Learning goals

  • Explain simple ciphers such as Caesar and Vigenere.
  • Compare symmetric and public key encryption.
  • Explain lossy and lossless compression.
  • Judge the impact of security and compression choices.

Quick memory hooks

  • Encryption: makes data unreadable without the right key.
  • Symmetric: same key to encrypt and decrypt.
  • Public key: paired public and private keys.
  • Compression: reduces file size.

Why this matters

Security protects privacy, identity, and business data. Compression can reduce bandwidth use and storage needs, but some compression methods permanently remove detail.

Encryption Basics

Simple ciphers show the basic idea of scrambling data

Simple ciphers are not strong enough for modern secure systems, but they help explain the central idea of encryption: plain text is transformed into cipher text using a rule or key.

Caesar cipher fixed shift Vigenere cipher repeating keyword shift

Caesar cipher

A Caesar cipher shifts each letter by the same number of places in the alphabet. For example, a shift of 3 changes A to D and CAT to FDW.

This is easy to understand, but it is also easy to break because there are only a small number of possible shifts.

Vigenere cipher

A Vigenere cipher uses a keyword so the shift changes from one letter to the next. That makes patterns less obvious than in a simple fixed-shift cipher.

It is stronger than a Caesar cipher, but it is still not secure enough for modern sensitive data.

System Encryption

Computer systems use key-based encryption to protect real traffic

Real systems need methods that scale beyond classroom ciphers. Unit 2 expects you to recognise the difference between symmetric key encryption and public key encryption, and the implications of using each.

Symmetric key encryption The same secret key is used to encrypt and decrypt the data.
Main strength It is efficient once both parties already have the key.
Main challenge The shared key itself has to be exchanged securely.
Public key encryption One key is public for encryption and a different private key is kept secret for decryption.
Main strength It helps solve the key-sharing problem because the public key can be distributed openly.
Main implication It is powerful for secure exchange, but usually more complex than simple symmetric use.

Why encryption matters in transmission

If intercepted traffic is encrypted, an attacker cannot easily read the original message without the necessary key. That is especially important when data crosses public or shared networks.

Compression

Compression reduces file size, but not always in the same way

Compression can make transmission faster and storage more efficient by reducing the number of bits needed to represent the data. The key exam difference is whether the original data can be rebuilt exactly.

Lossless no information lost Lossy some detail removed

Lossless versus lossy

Type What happens to the data Typical use Implication
Lossless The original data can be rebuilt exactly. Text files, documents, program files Safer when accuracy matters
Lossy Some information is removed permanently. Images, audio, video where small loss is acceptable Smaller files, but reduced quality

Why organisations compress data

Smaller files usually take less storage space and can often be sent more quickly. The trade-off is that lossy compression may remove detail which cannot be recovered later, so it is a poor choice for data that must stay exact.

Exam Traps

Common mistakes with encryption and compression

These questions often combine technical definitions with wider implications, so accuracy matters.

Assuming encryption compresses data

Encryption protects confidentiality. Compression reduces file size. They solve different problems.

  • Encryption is about privacy and security.
  • Compression is about efficiency and size.

Fix: State the purpose before naming the method.

Mixing up symmetric and public key encryption

Students often remember that keys are involved, but not whether the same key or a pair of keys is used.

  • Symmetric uses one shared secret key.
  • Public key uses a public and a private key.

Fix: Say "same key" or "key pair" explicitly.

Using lossy compression for exact data

Some files can tolerate lost detail, but documents, code, and many data files cannot.

  • Lossless suits files where every bit matters.
  • Lossy suits media where some reduction is acceptable.

Fix: Match the compression type to the data purpose.

4 marks Explain

Question 1

Explain why public key encryption is useful when two users need to exchange data securely but do not already share a secret key.

Show answer guide

The key idea is how the method helps with secure key exchange.

  • Public key encryption uses a public key and a private key.
  • The sender can use the public key without already sharing a secret key.
  • Only the matching private key can decrypt the data.
  • This helps protect data sent across shared or public networks.
5 marks Compare and justify

Question 2

A school needs to store text documents, photos, and training videos on a website. Compare lossless and lossy compression and justify which type is suitable for each kind of file.

Show answer guide

The best answers match the compression type to the importance of keeping every detail.

  • Text documents usually need lossless compression because exact data must be preserved.
  • Photos may use lossy compression if a small quality loss is acceptable in exchange for smaller file size.
  • Training videos also often use lossy compression because they are large and bandwidth matters.
  • Lossless preserves the original data exactly.
  • Lossy produces smaller files, but removed detail cannot be recovered.

Glossary

Quick glossary

Caesar cipher
A simple cipher that shifts letters by a fixed amount.
Symmetric key encryption
Encryption that uses the same key both ways.
Public key encryption
Encryption that uses a public and a private key.
Lossless compression
Compression that preserves the original data exactly.

Remember

Protect or reduce

  • Encryption protects confidentiality.
  • Compression reduces size.
  • Lossy compression saves space by sacrificing some detail.
