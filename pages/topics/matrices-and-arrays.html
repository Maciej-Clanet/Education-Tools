Overview

What you should know by the end of this lesson

Matrices and arrays both help organise related values, but they are not exactly the same idea. A matrix describes a grid of data arranged in rows and columns, while arrays give programs an indexed way to store and retrieve those values.

Learning goals

  • Define a matrix using rows, columns, and elements.
  • Explain the relationship between a matrix and an array.
  • Describe simple matrix operations accurately.
  • Link indexed access to how software stores the data.

Quick memory hooks

  • Matrix: a grid of values in rows and columns.
  • Element: one value at a chosen position.
  • 2D array: one common way to store a matrix.
  • Dimensions: the size of the matrix.

Where they appear

Matrices appear in seating plans, image pixels, score tables, and sensor grids. Arrays help programs keep those values in a structure that can be addressed by position.

Matrix

A matrix stores values in rows and columns

A matrix is a rectangular arrangement of values. Each value is an element and can be identified by its row and column position. In exam questions, that structure matters because it explains how values are grouped and which operations can be performed on them.

Row a horizontal line of values Column a vertical line of values Element one value at a chosen position
Why it works Two coordinates identify one exact value inside the grid.
Typical software use Pixel data, room layouts, score grids, and timetable blocks.
Common operations Addition, scalar multiplication, and sometimes matrix multiplication.
Key implication Dimensions matter, so not every pair of matrices can be combined.

Worked example

A seating grid for two rows and three columns can be written as this matrix: 

[ [A1, A2, A3],
  [B1, B2, B3] ]

The element in row 2, column 3 is B3. In a program using zero-based indexes, that same position would often be accessed with seats[1][2].

Matrix operations you should recognise

Operation Rule Exam point
Addition Both matrices must have the same dimensions. Combine matching positions element by element.
Scalar multiplication Multiply every element by the same number. Shape stays the same, but values change.
Matrix multiplication The number of columns in the first matrix must match the number of rows in the second. Do not treat this like simple position-by-position multiplication.

Addition

Add matrices by combining matching positions

Matrix addition works only when both matrices have the same dimensions. You add each element to the element in the same row and column, so the answer keeps the same shape as the originals.

Same dimensions rows and columns must match Match positions add like-for-like elements Same shape the result keeps the same layout

Scalar Multiplication

Multiply every element by the same scalar

A scalar is just one number. In scalar multiplication, that number multiplies every element in the matrix, so the values change but the rows and columns stay in the same arrangement.

One number the scalar multiplies every element Same shape the matrix keeps its dimensions Every cell changes multiply one by one

Matrix Multiplication

Multiply a row by a column, not just matching positions

Matrix multiplication follows a different rule from addition. The number of columns in the first matrix must match the number of rows in the second. Each result cell comes from multiplying along a row and down a column, then adding those products together.

Inner dimensions must match to make the multiplication valid Row x column one row combines with one column Outer dimensions decide the answer size

Worked visual example

This example uses a 2 x 2 matrix multiplied by another 2 x 2 matrix, so the result is also 2 x 2.

Matrix A
1 2 3 4
x
Matrix B
2 0 1 5
=
Result
4 10 10 20
First result cell (1 x 2) + (2 x 1) = 4
Second result cell (1 x 0) + (2 x 5) = 10
Answer size The outer dimensions are 2 rows by 2 columns, so the result is a 2 x 2 matrix.

Array Storage

Arrays let programs store matrix values with indexes

In computing, a matrix is often stored using arrays. A one-dimensional array uses one index, while a two-dimensional array uses a row index and a column index. That makes it easier for software to retrieve a chosen element directly.

1D array one position number 2D array row index plus column index Direct access jump to a chosen position quickly

Software view

Programs can use syntax such as scores[2][4] to reach one exact value in a table.

Hardware link

Indexed array access supports predictable addressing rather than scanning every value one by one.

Next step

The deeper memory layout rules for multi-dimensional arrays come next when you study memory order.

Worked storage example

A program storing points from two teams across three rounds could use this two-dimensional array: 

points = [
  [12, 9, 15],
  [10, 14, 11]
]

With zero-based indexing, points[1][0] means the first score in the second row. That is why a good answer links row and column positions to indexed storage.

Relationship in one sentence

A matrix is the logical grid of values, while a two-dimensional array is one common way a program stores that grid so each element can be accessed by index.

Exam Traps

Common mistakes students make with matrices and arrays

Most errors come from losing track of positions or using the wrong rule for an operation. These are the traps written questions often exploit.

Mixing up row and column positions

Students sometimes name the correct value type but put it in the wrong place because they reverse the coordinates.

  • Row is the horizontal line of values.
  • Column is the vertical line of values.
  • Programming indexes often start at 0, not 1.

Fix: State clearly whether you are using matrix positions or zero-based array indexes.

Ignoring dimension rules

Some answers try to add or compare matrices without checking whether they have matching dimensions.

  • Addition needs the same number of rows and columns.
  • Matching positions must represent equivalent data points.

Fix: Mention the shape of the matrix before explaining the operation.

Treating matrix multiplication like addition

Addition works element by element, but matrix multiplication follows a different rule and depends on inner dimensions.

  • Do not multiply only the matching positions.
  • The first matrix columns must match the second matrix rows.

Fix: If the question mentions multiplication, check the rule before describing the result.

4 marks Explain

Question 1

Explain why a two-dimensional array is a suitable way to store seat occupancy for a theatre with 6 rows and 10 seats in each row.

Show answer guide

A strong answer links the grid layout to indexed access in software.

  • The seating plan has a natural row-and-column structure.
  • Each seat can be stored as one element in a two-dimensional array.
  • The program can access a chosen seat directly using row and column indexes.
  • This keeps the data organised in the same layout as the real seating grid.
5 marks Explain and justify

Question 2

A school records weekly points for two classes in matrices of the same size, with one row per student and one column per week. Explain how the program could combine the matrices and why the dimensions must match.

Show answer guide

The best answers explain both the matrix rule and the data meaning behind it.

  • The program can add the matrices element by element.
  • Each position must represent the same student and the same week in both matrices.
  • If the dimensions match, every value has a corresponding value to combine with.
  • The result is another matrix with the same dimensions.
  • If dimensions do not match, some positions will not line up correctly so direct addition is invalid.

Glossary

Quick glossary

Matrix
A grid of values arranged in rows and columns.
Element
One individual value inside a matrix.
Dimension
The size of a matrix, described by its rows and columns.
Two-dimensional array
A structure that stores data using a row index and a column index.

Remember

Think in positions

  • A matrix is the grid itself.
  • A two-dimensional array is one storage method for that grid.
  • Dimension rules decide whether an operation is valid.
