Overview

What you should know by the end of this lesson

Computers do not store letters as visible shapes. They store numeric codes that software interprets as characters, symbols, and control instructions.

Learning goals

  • Explain why character sets are needed.
  • Describe the basic features of ASCII.
  • Explain why Unicode is widely used.
  • Link encoding choices to compatibility and storage.

Quick memory hooks

  • Character set: agreed code mapping.
  • ASCII: smaller, older standard.
  • Unicode: much wider character coverage.
  • Encoding: how text becomes stored data.

Foundations

Character sets let systems agree what each code means

A character set is a standard list that matches numeric codes to letters, digits, punctuation, symbols, and control characters. If two systems use different mappings, the same stored value can be displayed as the wrong character.

Text is data Characters must be converted into binary before a computer can store or transmit them.
Agreed meaning The standard makes sure code 65, for example, means the same thing to both sender and receiver.
Control codes Some values are not visible letters at all. They may mean line break, tab, or another command.
Compatibility issue Mismatched encodings can create garbled text or missing symbols.

ASCII

ASCII is a small, widely recognised character standard

ASCII uses a limited number of codes to represent common English letters, digits, punctuation, and control characters. It is often described as a 7-bit standard, giving 128 possible codes.

7-bit standard 128 codes Good for basic Latin text Limitation small character range

Why ASCII still matters

ASCII appears in many exam questions because it is simple and historically important. It helps explain how text encoding works before moving to larger standards.

Unicode

Unicode supports a far wider range of languages and symbols

Unicode was designed so modern systems can represent characters from many languages, plus mathematical symbols, currency signs, and more. It solves the problem of ASCII being too limited for global computing.

Why Unicode is important

A multilingual website, messaging app, or document system needs more than the small ASCII set. Unicode allows text from different writing systems to be handled more consistently across software and devices.

Main trade-off

Supporting a much wider range of characters can increase storage and processing demands compared with a smaller encoding, but the compatibility benefits are usually worth it.

Implications

Encoding choices affect compatibility, storage, and user experience

Exam Traps

Common mistakes with text representation

Calling ASCII a language

ASCII is a coding standard, not a spoken or written language.

Fix: Describe it as a character set or encoding standard.

Assuming Unicode means one language

Unicode exists to support many languages and symbol sets, not a single one.

Fix: Link Unicode to international or multilingual text.

Ignoring compatibility

Encoding is not just about storage. It affects whether systems can exchange and display text correctly.

Fix: Mention mismatched codes causing garbled characters.

4 marks Explain

Question 1

Explain why computers need character sets instead of storing letters directly as visible symbols.

Show answer guide
  • Computers store data as binary values.
  • Characters need a standard code so their binary value has an agreed meaning.
  • Software uses the code to display the correct letter or symbol.
  • Without a shared character set, text could be interpreted incorrectly.
6 marks Compare and justify

Question 2

Compare ASCII and Unicode for an international messaging app, then justify which standard is more suitable.

Show answer guide
  • ASCII has a small character range.
  • Unicode supports far more languages and symbols.
  • Unicode may use more storage than a smaller encoding.
  • An international messaging app needs broad language support, so Unicode is the stronger justified choice.

Glossary

Quick glossary

Character set
An agreed mapping between codes and characters.
ASCII
A smaller character standard often described as 7-bit.
Unicode
A much wider standard for characters and symbols.
Control character
A code that gives an instruction rather than visible text.

Remember

Same code, same meaning

  • Character sets let systems interpret stored text consistently.
  • ASCII is simple but limited.
  • Unicode is stronger for global modern computing.
