Overview

What you should know by the end of this lesson

Boolean logic reduces decisions to two states: true or false, 1 or 0, on or off. That makes it useful for system control, decision making, and explaining how inputs combine to produce an output.

Learning goals

  • Explain what Boolean logic represents.
  • Use AND, OR, and NOT correctly.
  • Read and complete simple truth tables.
  • Apply logic to short system scenarios.

Quick memory hooks

  • AND: both conditions must be true.
  • OR: at least one condition is true.
  • NOT: flips the value.
  • Truth table: all possible inputs.

Why it matters

Exam questions often describe alarms, sensors, access control, or control systems in words. Boolean logic gives you a precise way to turn that wording into a clear rule for the output.

Operators

Logic operators describe how conditions combine

Each operator has a simple rule. The key is to read the wording carefully and match it to the correct logic relationship.

AND both inputs needed OR either input is enough NOT reverse the state
AND Output is true only when every required condition is true.
OR Output is true when at least one condition is true.
NOT Output becomes the opposite of the original input.
Boolean values Systems usually store the result as true or false, or 1 or 0.

From words to logic

If a door alarm sounds only when the system is armed and the door opens, the rule is: 

alarm = armed AND doorOpen

If a warning light turns on when the battery is low or the temperature is too high, the rule is: 

warning = lowBattery OR highTemperature

What NOT changes

NOT is often used when the sensor signal is the opposite of the condition you want to test. For example, if doorClosed is true, then NOT doorClosed means the door is open.

Truth Tables

Truth tables show every possible input combination

A truth table is a reliable way to test logic because it lists all possible input states and the output for each one. With two inputs there are four rows: 00, 01, 10, and 11.

Core patterns to remember

Input A Input B A AND B A OR B
0 0 0 0
0 1 0 1
1 0 0 1
1 1 1 1

Worked NOT example

If safeMode is 1, then NOT safeMode is 0. If safeMode is 0, then NOT safeMode is 1. The operator always flips the state.

How to read exam wording

  • “Only when both” usually signals AND.
  • “If either” usually signals OR.
  • “When not” or “unless” often signals NOT.
  • A truth table is often the easiest way to prove your answer.

Application

Boolean logic helps model data flow and solve system problems

In a real system, sensor or user inputs feed into a logical rule. The result decides whether an output should happen. That makes Boolean logic useful for security, automation, and process control.

Access control example

A secure door opens only when a valid passcode is entered and the account is active: 

doorOpen = validCode AND accountActive

If either condition is false, the door stays locked.

Alarm example

A home alarm may trigger when a window sensor or a door sensor reports a breach while the system is armed: 

alarm = armed AND (windowOpen OR doorOpen)

This shows how logic can combine conditions to reflect the real behaviour of a system.

How to build a logic answer

Step What to do Why it helps
1 Identify the inputs Clarifies what the system is checking
2 Spot the condition words Links the scenario to AND, OR, or NOT
3 Write the logic rule Turns the description into a precise expression
4 Test it with a truth table Checks whether the output matches the scenario

Exam Traps

Common mistakes with Boolean logic

Most errors happen because the wording in the scenario is read too quickly or because students describe the result but not the logic behind it.

Mixing up AND and OR

Students often write OR when a system needs every condition to be true.

Fix: Ask whether one input is enough or whether all inputs are required.

Forgetting what NOT flips

NOT changes the state of the input or condition it applies to.

Fix: Rewrite the statement in plain English, for example “NOT locked” means “unlocked”.

Skipping the truth table

Some answers sound reasonable but are not checked against all input combinations.

Fix: Build the table when the question asks you to justify or verify the logic.

Alarm control logic

A building alarm should sound only when the system is armed and a motion sensor detects movement. Explain how Boolean logic can represent this rule.

Answer guidance

A strong answer should explain that the two inputs are armed and motionDetected, and that the output should only become true when both inputs are true, so the expression is: 

alarm = armed AND motionDetected
  • Identify the inputs clearly.
  • State that AND needs both conditions to be true.
  • Link the expression to the alarm output.
  • Use accurate Boolean vocabulary.
6 marks Apply

Truth table reasoning

A cooling fan turns on when the temperature is high or when a manual override switch is pressed. Explain how you would use a truth table to test this logic.

Answer guidance

A strong answer should say that the expression is: 

fan = highTemperature OR manualOverride

The truth table should include the four possible input combinations and show that the fan is only off when both inputs are false.

  • State the correct OR relationship.
  • Explain that two inputs create four rows.
  • Show that 00 gives 0.
  • Show that 01, 10, and 11 give 1.
  • Link the output back to the scenario.
