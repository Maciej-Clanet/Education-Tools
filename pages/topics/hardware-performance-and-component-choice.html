Overview

What you should know by the end of this lesson

In hardware questions, performance and choice are linked. A fast component may not be the best option if it is too expensive, incompatible, power hungry, or difficult to implement.

Learning goals

  • Explain how hardware features affect performance.
  • Compare component choices using technical evidence.
  • Consider cost, compatibility, accessibility, and security.
  • Make balanced recommendations for a scenario.

Quick memory hooks

  • Performance: how well it runs.
  • Compatibility: will it work together?
  • Efficiency: useful output per resource.
  • Justify: give a reason, not a label.

Exam habit to build

Use a chain of reasoning. For example: "An SSD improves boot time, so productivity rises because users spend less time waiting for programs to load."

CPU

CPU performance affects how quickly instructions can be processed

CPU choice matters when the system runs demanding software, supports many tasks at once, or needs fast response to user actions. In this A1 lesson, focus on visible component choice rather than the deeper instruction-cycle detail covered later in B2.

Clock speed GHz Higher can help single-threaded tasks.
Cores and threads count More can help multitasking and parallel work.
Cache MB Fast on-chip memory reduces repeated RAM access.
Power and heat W Thermal limits affect sustained speed.
CPU factor What it means Best performance usually means Scenario link
Clock speed Cycles per second, commonly measured in GHz. Higher speed for tasks that rely on fewer cores. Useful for responsive office apps or some games.
Core count Number of processing cores available. More cores where software can split work. Useful for rendering, virtual machines, and heavy multitasking.
Cache Small fast memory close to the CPU. Enough cache to reduce slower memory access. Helps repeated access to active instructions and data.
Thermal design How much heat and power the CPU must manage. A suitable balance of speed, cooling, and efficiency. Important in laptops, mobiles, and compact PCs.

Memory

RAM affects how much active work a system can handle smoothly

RAM is working memory. If there is not enough, the system may rely on slower storage, causing applications to lag or switch slowly.

Limited RAM

Active programs spill into slower storage.

Enough RAM

More active work stays in fast memory.

Capacity Measured in GB. More capacity helps with multitasking, large files, browser tabs, virtual machines, and creative software.
Speed and bandwidth Faster RAM can move more data per second, but the benefit depends on CPU, motherboard, and workload.
Compatibility RAM must match the motherboard type, supported capacity, and speed limits.
Upgrade value Extra RAM is useful only if the current system is actually running short during normal use.

Storage

Storage performance affects loading, saving, transfer, and capacity

Storage choice should match how data is used. A system drive, media editing drive, backup drive, and archive drive can have different priorities.

SSD

Best when responsiveness, access time, shock resistance, and quiet operation matter.

HDD

Best when very large capacity and lower cost per GB matter more than speed.

External storage

Best for transfer, backup, or portable working when the connection is fast enough.

Storage factor Common unit Best performance usually means Why it matters
Capacity GB or TB Enough space for OS, apps, files, and growth. Prevents constant clearing, archiving, or external transfers.
Read/write speed MB/s or GB/s Higher speeds for large file loading and saving. Important for video, games, datasets, and system drives.
Access time ms Lower delay before data starts transferring. Improves booting, opening apps, and random access.
Reliability and endurance TBW, hours, or rating Suitable lifespan for the workload. Important for systems with heavy writes or critical data.

Graphics

Graphics hardware affects visual, creative, gaming, and parallel workloads

Not every user needs a dedicated GPU. The right choice depends on whether the workload needs advanced rendering, video editing, simulation, 3D design, multiple displays, or GPU acceleration.

Integrated graphics

Lower cost and power use. Suitable for office work, browsing, and basic media.

Dedicated GPU

More visual processing power and often dedicated VRAM. Suitable for creative or 3D work.

GPU capability Affects rendering, visual effects, 3D modelling, games, and software that can use GPU acceleration.
VRAM Graphics memory, measured in GB. More VRAM helps with high resolutions, large textures, and complex scenes.
Display support The GPU and ports must support the required number of screens, resolution, and refresh rate.
Power and cooling High-performance GPUs may need more power, larger cases, and better cooling.

Display

Display performance affects clarity, smoothness, and readability

Display choice is part of hardware performance because the screen controls how clearly, smoothly, and comfortably users see output.

Refresh rate smoother motion Response time less blur or ghosting Brightness readable in bright spaces Contrast clearer dark and light detail
Display factor Description Common unit Best performance usually means
Refresh rate How many times the display updates each second. Hz Higher refresh rate for smoother motion.
Response time How quickly pixels change from one state to another. ms Lower response time to reduce blur or ghosting.
Brightness How much light the display can emit. cd/m2 Higher brightness for bright rooms or visibility needs.
Contrast ratio Difference between the darkest black and brightest white. ratio, such as 1000:1 Higher contrast for clearer image detail.
Resolution Number of pixels shown on the screen. pixels, such as 1920 x 1080 Higher resolution for sharper detail, if the system can drive it.

Printer

Printer performance depends on speed, first-page time, resolution, and workload

Printer choice depends on what is printed, how often it is printed, how quickly users need output, and how much detail is required.

Print speed ppm or ipm First-page-out time seconds Resolution dpi Duty cycle pages/month
Printer factor Description Common unit Best performance usually means
Printing speed How many pages or images can be printed in a given time. ppm or ipm Higher speed for busy offices or batch printing.
First-page-out time How long the first page takes after a print job starts. seconds Lower time for users who print short jobs often.
Maximum printing resolution How much detail the printer can place on the page. dpi Higher resolution for detailed images or design proofs.
Monthly duty cycle The volume a printer is designed to handle per month. pages/month Higher duty cycle for heavy, shared office use.

Network And I/O

Network and connection choices affect shared work and peripheral speed

A fast computer can still feel slow if its network connection, wireless signal, or external ports cannot move data quickly enough for the task.

Bandwidth Measured in Mbps or Gbps. Higher bandwidth helps when moving large files, backing up, or streaming high-quality media.
Latency Measured in ms. Lower latency improves interactive tasks such as video calls, cloud desktops, and online applications.
Connection type Ethernet, Wi-Fi, USB, HDMI, DisplayPort, and other interfaces must support the required speed and features.
Reliability and availability Shared systems may need dependable connections, spare ports, or fallback options to reduce downtime.

Choice Factors

Choosing hardware means balancing technical performance with practical constraints

In Unit 2, strong answers often compare several decision factors before making a recommendation.

User experience Ease of use, performance, availability, accessibility
Technical fit Specifications, compatibility, security
Practical fit Cost, efficiency, implementation, productivity
Factor What it asks Example impact
User experience Will it be easy, fast, available, and accessible for the user? A larger high-contrast display may help a user work more comfortably.
User needs What must the system actually do? A school office PC does not need the same GPU as a 3D workstation.
Specifications Do the measurable features meet the task? A design monitor may need high resolution and good contrast, not only a large screen.
Compatibility Will the part work with the existing system? A motherboard may not support a newer processor socket.
Cost Is the improvement worth the extra money? A premium component may not give enough benefit for basic tasks.
Efficiency How much energy or heat does the system use? A low-power device may be better for battery life or running costs.
Implementation How difficult is the change to install, test, migrate, and roll out? New hardware may need timescales, testing, user training, and data migration.
Productivity Will users complete work faster or with fewer errors? More RAM may reduce delays when staff keep several applications open.
Security Does the choice protect data and system access? Encrypted storage, TPM support, locks, or controlled ports may reduce risk.

Judgement

Good recommendations explain both the benefit and the trade-off

A recommendation is strongest when it recognises what the user gains and what they may give up.

SSD faster, but costs more per GB More RAM better multitasking Low-power chip longer battery life

Reception desk

Prioritise reliability, quick startup, dual display support, and accessibility over gaming-level graphics.

Design studio

Prioritise RAM, fast storage, GPU capability, colour/detail display quality, and suitable printer resolution.

Shared print room

Prioritise print speed, low first-page-out time, duty cycle, running costs, security, and network compatibility.

Worked comparison

If a business replaces old HDDs with SSDs, staff may gain faster boot times and quicker file access, improving productivity. However, the upgrade increases cost and may need time for imaging, migration, and testing before rollout.


What balanced judgement sounds like

Try writing: "Although the component is more expensive, it is justified because the improvement in performance saves staff time every day." That is usually stronger than simply saying "It is better."

Exam Traps

Common mistakes with hardware performance and choice

These questions often lose marks because students focus on one factor and ignore the wider decision.

Assuming the most powerful part is always best

That ignores cost, efficiency, and real user need.

Fix: Match the component level to the actual workload.

Forgetting compatibility

A new part may not fit or work with the current system.

Fix: Mention whether the system can support the upgrade.

Giving an unsupported judgement

"Use an SSD" is not enough without a reason.

Fix: Say what improves and why that matters in the scenario.

Quoting a number without its unit

"A high brightness screen" is weaker than naming brightness in cd/m2 or print resolution in dpi.

Fix: Use units such as Hz, ms, cd/m2, dpi, ppm, and pages/month where the factor needs a measurement.

Ignoring rollout work

Hardware changes may need testing, migration, training, and downtime.

Fix: Mention implementation when a choice affects many users or existing systems.

5 marks Explain

Question 1

Explain how replacing a hard disk drive with a solid-state drive can improve user experience and productivity.

Show answer guide
  • SSDs usually give faster boot and loading times.
  • Files and programs open more quickly.
  • Users spend less time waiting.
  • This can improve day-to-day productivity.
  • The answer may also note that SSDs are more durable because they have no moving parts.
6 marks Discuss

Question 2

A small business wants to upgrade old office computers. Discuss the factors that should affect the final hardware choice.

Show answer guide
  • User needs should come first because office work may not require specialist high-end hardware.
  • Compatibility matters if the business wants to reuse existing systems or peripherals.
  • Cost must be justified against the benefit gained.
  • Implementation affects downtime, setup, data migration, and testing.
  • Efficiency and energy use may affect running costs.
  • Security and productivity improvements should also be considered in the final decision.
8 marks Analyse

Question 3

A college design room needs new monitors and a shared printer for graphics students. Analyse which display and printer performance factors should affect the hardware choice.

Show answer guide
  • Monitor resolution and contrast affect how clearly students can see detail in design work.
  • Brightness, measured in cd/m2, matters if the room is bright or users need high visibility.
  • Refresh rate and response time matter more for motion, animation, or video work than for simple documents.
  • Printer resolution, measured in dpi, affects detail and print quality for design proofs.
  • Print speed and first-page-out time affect how long students wait for work.
  • Monthly duty cycle should match shared classroom use so the printer is not overloaded.

Glossary

Quick glossary

Compatibility
Whether hardware parts or systems work together.
Specification
A measurable hardware feature such as GHz, GB, Hz, dpi, or ppm.
Refresh rate
How often a display updates each second, measured in Hz.
Response time
How quickly display pixels change, usually measured in ms.
Brightness
Display light output, often measured in cd/m2.
Duty cycle
The print volume a printer is designed to handle each month.
Efficiency
How effectively a system uses power or resources.
Implementation
The practical process of installing and rolling out change.
Productivity
How effectively users can complete their work.

Remember

Balance the judgement

  • Faster is not automatically better.
  • Link the hardware improvement to a real user benefit.
  • Use units when comparing measurable specifications.
  • Strong recommendations mention trade-offs and rollout.
