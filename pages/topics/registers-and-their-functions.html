Overview

What you should know by the end of this lesson

Registers are very small, very fast storage locations inside the CPU. Exam questions often test whether you can name the correct register, explain what information it holds, and connect that function to the instruction cycle or overall CPU performance.

Learning goals

  • Define a register and explain why it is faster than main memory.
  • Describe the role of general purpose registers.
  • Name the key special registers used in Unit 2.
  • Explain how register use supports processor performance.

Quick memory hooks

  • Register: tiny, fast storage inside the CPU.
  • MAR: memory address to visit.
  • MDR: data being transferred.
  • PC: address of the next instruction.

Where they appear

Registers matter whenever the CPU fetches, decodes, executes, stores temporary values, or communicates with main memory.

Register Basics

Registers give the CPU fast working storage

A register is a storage location inside the CPU itself. Because registers are built into the processor, they can be accessed much faster than RAM. That speed matters because the CPU is constantly reading instructions, holding temporary values, and moving data.

Small hold only a limited amount of data Fast quicker than accessing main memory Temporary used while processing is happening
Why they matter The CPU needs immediate access to important values without waiting for slower memory.
General purpose registers Hold temporary data and intermediate values while a program is running.
Special purpose registers Hold very specific information such as addresses, instructions, or transferred data.
Performance link Fast register access reduces how often the CPU must pause for slower memory operations.

General purpose registers

A general purpose register can hold temporary values that the CPU is currently using. For example, it may store part of a calculation result, a copied value, or data needed for the next processing step.

In exam answers, the key point is flexibility: a general purpose register is not tied to one single task like the MAR or program counter.

Special Registers

Each special register has a precise job

Pearson Unit 2 expects you to recognise the accumulator, instruction register, memory address register, memory data register, and program counter. Strong answers explain what each one holds, not just its name.

Register Main function Typical exam wording
Accumulator Stores intermediate arithmetic or logic results. Temporary result of a calculation
Instruction Register Holds the current instruction while it is being decoded or executed. Current instruction
Memory Address Register (MAR) Holds the address in memory that the CPU wants to access. Address to read from or write to
Memory Data Register (MDR) Holds the data being transferred to or from memory. Data moving between CPU and memory
Program Counter (PC) Holds the address of the next instruction to fetch. Address of next instruction

During fetch

The program counter gives the next instruction address, and the MAR uses that address to reach memory.

During transfer

The MDR carries the data or instruction that is moving between memory and the CPU.

During processing

The instruction register holds the current instruction while the accumulator or general purpose registers support the work.

Worked instruction-cycle example

Suppose the CPU is about to fetch the next instruction. The program counter holds the address of that instruction. The address is copied into the MAR, memory is accessed, and the fetched instruction moves back through the MDR. The instruction is then placed in the instruction register so the CPU can decode it.

If the instruction involves arithmetic, the accumulator may then hold the intermediate result while the calculation continues.

Comparison

Choose the right register explanation for the question

The most common weakness in register questions is naming a register without saying what it holds. A stronger answer explains the type of information stored and where it fits in processing.

Question focus Best register answer Why it fits
Next instruction address Program counter It stores where the CPU fetches from next.
Address being accessed now MAR It points to the exact memory location in use.
Data being transferred MDR It carries the data or instruction in transit.
Instruction currently being decoded Instruction register It stores the current instruction inside the CPU.
Temporary result of a calculation Accumulator or general purpose register It keeps a working value close to the ALU.

Mention performance when...

The question asks why registers matter or why CPUs can work so quickly on active data.

Mention the instruction cycle when...

The question asks how the CPU fetches, stores, or processes instructions and data.

Exam reminder

MAR means address, MDR means data. Those two are often tested together because students mix them up.

Exam Traps

Common mistakes students make with registers

Register questions are often easy to lose marks on because several names sound similar. These are the mix-ups that come up most often in written answers.

Mixing up MAR and MDR

Students often say both hold "memory stuff" without separating address from data.

  • MAR holds the memory location.
  • MDR holds the value moving to or from that location.

Fix: Use the exact words address and data in your answer.

Confusing the program counter with the current instruction

The program counter does not store the whole instruction. It stores where the next instruction is located.

  • PC holds the next instruction address.
  • Instruction register holds the current instruction.

Fix: Link the PC to fetching and the instruction register to decoding or execution.

Forgetting why registers improve performance

Some answers list register names correctly but never explain why the CPU uses them.

  • Registers are inside the CPU and very fast to access.
  • They reduce constant trips to slower main memory.

Fix: Add one sentence about speed or reduced memory access when the question allows it.

4 marks Explain

Question 1

Explain the different roles of the MAR and MDR when the CPU fetches data from main memory.

Show answer guide

A strong answer clearly separates address from data.

  • The MAR stores the memory address that the CPU wants to access.
  • Main memory is read using that address.
  • The MDR stores the data that is returned from memory.
  • So the MAR points to the location, while the MDR carries the value.
5 marks Explain and justify

Question 2

Explain why registers are important for CPU performance, using at least one general purpose register and one special purpose register in your answer.

Show answer guide

The best answers link register function to speed and CPU workflow.

  • Registers are inside the CPU, so they are much faster to access than RAM.
  • A general purpose register can keep a temporary working value ready for immediate use.
  • A special register such as the PC, MAR, or MDR supports fast instruction and data movement.
  • This reduces delays caused by repeatedly accessing slower main memory.
  • Faster access to active values helps the CPU complete the instruction cycle more efficiently.
