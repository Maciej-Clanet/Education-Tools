Overview

The kernel is the operating system control centre

The kernel is the central part of an operating system. Programs ask for services, but the kernel decides how CPU time, memory, backing storage, and hardware access are actually handled.

Applications Browser, game, editor, media player
system calls
Kernel Controls protected work and shares resources
controlled access
Hardware CPU, RAM, disk, network card, graphics card

Learning goals

  • Define the kernel as the core part of the operating system.
  • Explain how the kernel controls program execution, interrupts, modes, memory, multitasking, disk access, file systems, and drivers.
  • Compare simple file systems such as FAT32 with more sophisticated systems such as NTFS.
  • Use applied examples instead of vague phrases such as "the OS runs everything".

Quick memory hooks

  • Load: start software and allocate resources.
  • Protect: separate user mode and kernel mode.
  • Share: divide CPU time and RAM between tasks.
  • Connect: use drivers and file systems to reach hardware.

Why this topic feels dry

Most kernel work is invisible when a computer is working well. A useful way to revise it is to picture the kernel as a traffic controller: programs request work, hardware sends signals, and the kernel decides what is allowed to happen next.

The exam sentence pattern

Use: "The kernel manages [resource or task] by [action], so that [benefit]." For example, "The kernel manages RAM by allocating memory to running programs and reclaiming it when they close, so programs do not overwrite each other and memory can be reused."

Program Execution

Starting a program is a managed life cycle

Program execution is not just "open the app". The kernel helps load the software, allocate resources, run it, and terminate it cleanly when it is finished.

Example

When a student opens image-editing software, the kernel helps load the program, gives it memory for the image and editing tools, schedules CPU time while filters run, and releases the memory and file handles when the program closes.

Interrupts And Modes

Interrupts get attention; modes protect powerful operations

Interrupts let components request servicing by the CPU. Modes decide whether code is running as an ordinary application or as trusted low-level operating system code.

Memory management and multitasking make concurrent work possible

The kernel allocates RAM to running programs and reuses RAM when programs end. It also shares CPU time and peripheral time so several programs can appear to work at the same time.

Storage And Devices

Disk access, file systems, and drivers make hardware usable

The kernel controls how data is read from and written to backing storage. File systems organise the stored data, while device drivers let the operating system communicate with hardware.

System Management

The same control ideas support networking and security

The kernel is the core controller, but operating system management also includes network communication, access control, permissions, and protecting shared resources from unauthorised or unsafe actions.

Networking

The OS manages network interfaces, connections, and the software support needed to send and receive data.

Security

Accounts, permissions, and protected modes help prevent unauthorised access or damaging actions.

Access rights

Users and programs can be given different levels of access to files, devices, and settings.

Stability

Controlled access reduces the chance of one program crashing or damaging the whole system.

Scenario link

On a shared college computer, the operating system must let a student log in, connect to the network, save work to the right location, use devices through drivers, and block changes to protected system files.

Exam Traps

Common mistakes with kernel functions

Kernel questions usually reward precise control language. Avoid answers that only describe what the user sees on screen.

Calling the kernel the whole operating system

The kernel is central, but it is not every part of the OS.

Fix: Describe it as the core part that controls resources and protected tasks.

Explaining the app instead of the OS

"The browser opens the web page" does not explain kernel management.

Fix: Mention loading, RAM allocation, CPU scheduling, file access, or device communication.

Thinking interrupts are only errors

Interrupts can be normal requests, such as a mouse click or a timer event.

Fix: Say that an interrupt requests CPU servicing, then give a hardware example.

Forgetting FAT32 and NTFS detail

File system questions often need more than "stores files".

Fix: Link FAT32 to simple removable storage and NTFS to larger devices, access rights, recovery, or encryption.

Question 1

Describe two ways the kernel is involved when a user opens and then closes an application.

Show answer guide
  • The kernel helps load the program from storage into memory.
  • It allocates resources such as RAM and processor time.
  • It manages the running process while the application is active.
  • When the application closes, the kernel terminates the process and releases resources for reuse.
6 marks Explain

Question 2

Explain how interrupts and modes help an operating system respond to hardware while protecting the computer system.

Show answer guide
  • Hardware such as a mouse, keyboard, disk, or network card can send an interrupt when it needs attention.
  • The CPU can pause or switch from the current task so the operating system can service the request.
  • User mode restricts ordinary application code so it cannot directly perform dangerous low-level actions.
  • Kernel mode lets trusted OS code carry out privileged operations such as accessing memory or devices.
  • This helps the system respond to events while reducing the risk of an application damaging the system.
  • A strong answer uses a specific example, such as a mouse click or disk read.
8 marks Discuss

Question 3

A college is setting up shared computers and removable USB drives for students. Discuss why file systems and device drivers are important in this scenario.

Show answer guide
  • File systems control how data is stored, addressed, organised, saved, and retrieved.
  • FAT32 may be suitable for simple removable USB flash drives where security is not required.
  • NTFS may be better for internal shared computer drives because it can support access rights, recovery from disk errors, and encryption.
  • Access rights can help stop students viewing or changing files they should not access.
  • Device drivers let the OS communicate with hardware such as graphics cards, printers, network adapters, or USB controllers.
  • Correct drivers help devices use their features and performance properly.
  • A balanced answer links storage organisation, security, compatibility, and hardware communication to the college scenario.

Glossary

Quick glossary

Kernel
The core part of the operating system that controls resources and system tasks.
Interrupt
A signal that requests attention from the CPU.
User mode
A restricted mode used by ordinary applications.
Kernel mode
A privileged mode used for low-level OS operations.
File system
The method used to organise, address, save, and retrieve files on storage.
Device driver
Software that allows the OS to communicate with hardware.

Remember

Name the control

  • Programs request work; the kernel controls access.
  • Interrupts ask for CPU attention.
  • User mode is restricted; kernel mode is privileged.
  • FAT32 is simple; NTFS has stronger management features.
