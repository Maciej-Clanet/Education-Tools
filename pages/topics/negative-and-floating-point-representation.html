Overview

What you should know by the end of this lesson

Unsigned binary only handles positive values and zero. Real computer systems also need to represent negative numbers, fractions, and values with a huge range, so other schemes are used.

Learning goals

  • Explain how negative values can be represented.
  • Describe the role of two's complement.
  • Outline the main idea behind floating point.
  • Identify precision limits and rounding issues.

Quick memory hooks

  • Unsigned: no negative values.
  • Two's complement: common negative format.
  • Floating point: sign, exponent, significand.
  • Trade-off: range versus exactness.

Negative Values

Computers need a way to show values below zero

One simple idea is a sign bit, where the leftmost bit shows positive or negative. In practice, two's complement is widely used because it makes arithmetic easier for the CPU.

Method Main idea Weakness or strength
Sign and magnitude One bit shows sign, the rest show size Simple idea but less convenient for arithmetic
Two's complement Invert the positive value and add 1 Efficient for binary arithmetic

Worked example for two's complement

In 8 bits, positive 5 is 00000101. To represent negative 5 in two's complement, invert the bits to get 11111010, then add 1. The result is 11111011.

Floating Point

Floating point stores a value using scale as well as digits

Floating point works like scientific notation. Instead of storing every digit in one fixed place, the system stores a sign, an exponent, and a significand or mantissa.

Sign positive or negative Exponent how far the point moves Significand the main digits

Worked idea

Decimal scientific notation might write 3500 as 3.5 x 10^3. A binary floating-point system uses the same principle with powers of 2. For example, 1101.01 can be normalised to 1.10101 x 2^3.

Precision

Floating point gives range, but not always exact values

Because the number of bits is limited, some real numbers cannot be stored exactly. This can lead to rounding, loss of precision, or overflow when a value is too large for the available format.

Precision How many significant digits or bits the system can keep.
Range How large or small a value can be while still fitting in the format.
Rounding The nearest representable value may be stored instead of the exact one.
Overflow The value is too large for the available number of bits.

Compare

Choose the representation by the type of value you need

Representation Useful for Main trade-off
Unsigned binary Non-negative whole numbers No negative values
Two's complement Negative and positive integers Bit width limits the range
Floating point Fractions and a large value range Possible rounding and precision loss

Exam Traps

Common mistakes with negative and floating point values

Forgetting the bit width

A binary pattern only has meaning inside a stated size such as 8 bits or 16 bits.

Fix: Keep the full bit pattern and do not drop leading bits.

Assuming floating point is always exact

Some decimal fractions cannot be represented perfectly in a limited binary format.

Fix: Mention rounding or precision when asked about limitations.

Mixing methods together

Sign-and-magnitude rules and two's complement rules are not interchangeable.

Fix: State which representation method the question is using before you work the answer out.

Glossary

Quick glossary

Two's complement
A common method for representing negative integers.
Exponent
The part that scales a floating-point value.
Significand
The main digits of a floating-point value.
Overflow
A value that is too large to fit in the format.

Remember

Range versus exactness

  • Two's complement helps with negative whole numbers.
  • Floating point helps with scale and fractions.
  • Greater range can come with precision limits.
