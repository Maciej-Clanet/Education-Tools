Overview

What you should know by the end of this lesson

The instruction cycle is the repeated process the CPU uses to run a program. In Unit 2, strong answers explain the stages in order and name what the main registers hold while the cycle is happening.

Learning goals

  • Define the instruction cycle clearly.
  • Explain what happens during fetch, decode, and execute.
  • Name the key registers linked to the cycle.
  • Apply the sequence to short exam scenarios.

Quick memory hooks

  • PC: address of the next instruction.
  • MAR: address that memory will be accessed at.
  • MDR: instruction or data moving to or from memory.
  • IR: current instruction being decoded or executed.

Where it appears

Every stored-program CPU depends on the instruction cycle. When you explain any processor action, you are usually describing one part of this repeated loop.

The Cycle

Fetch, decode, and execute happen again and again

Programs are stored as instructions in memory. The CPU works through them one at a time, repeating the same broad sequence so each instruction can be located, understood, and carried out.

Fetch get the next instruction from memory Decode work out what the instruction means Execute carry out the required action
Stage What the CPU is doing Why it matters
Fetch The CPU gets the next instruction from main memory. No instruction can run until it has been brought into the CPU.
Decode The control unit interprets the opcode and required action. The CPU must know whether to add, compare, jump, store, or do something else.
Execute The processor carries out the instruction, often using the ALU or memory access. This is where the actual work of the program happens.

Why exam questions like this topic

The instruction cycle links together registers, memory access, control signals, and processor performance. It is a good test of whether you understand how separate CPU concepts fit together.

Fetch Stage

The fetch stage moves the next instruction into the CPU

Fetch questions are often about register roles. The key is to say what each register holds at that moment rather than listing names without explanation.

Step Register action Result
1 The program counter holds the address of the next instruction. The CPU knows which memory location it needs next.
2 That address is copied into the MAR. Main memory can be accessed at the correct location.
3 The instruction is fetched from memory through the MDR. The instruction enters the CPU.
4 The instruction is placed in the instruction register. The control unit can decode it.
5 The program counter is increased unless a branch changes it. The CPU is ready to fetch the next instruction later.

PC

Stores the next instruction address, not the full instruction itself.

MAR and MDR

MAR points to the memory location. MDR carries the value or instruction.

Instruction register

Holds the current instruction once it has been fetched into the CPU.

Worked fetch example

If the next instruction is stored at memory address 205, the program counter may hold 205. That value is copied into the MAR, memory is read, and the instruction returns through the MDR. It is then loaded into the instruction register so the CPU can decode it.

Decode and Execute

After fetch, the CPU works out the meaning and performs the action

The control unit decodes the instruction so the processor knows which action is required. Execution may mean an arithmetic calculation, a comparison, a store to memory, or a jump to a new instruction address.

Decode The control unit interprets the opcode and prepares the control signals needed.
Execute The CPU carries out the required action, often using the ALU, registers, or memory access.
Accumulator link Arithmetic or logic work may place an intermediate result in the accumulator.
Branch link Some instructions change the usual sequence by loading a new value into the program counter.

Why decode is separate from execute

The CPU must first understand what the instruction means before it can carry it out. Saying "the CPU fetches and does it" skips an important exam mark because decoding is its own stage.

Exam Traps

Common mistakes students make with the instruction cycle

Instruction-cycle questions often look simple, but weak answers usually lose marks on register detail or the order of the stages.

Mixing up MAR and MDR

Students often describe both as if they do the same job.

  • MAR holds the address to access.
  • MDR holds the data or instruction being transferred.

Fix: Use the exact words address and data.

Forgetting decode

Some answers jump straight from fetch to execution.

  • The CPU must interpret the opcode first.
  • Decode is a separate part of the cycle.

Fix: Always state the full sequence as fetch, decode, execute.

Confusing the PC with the current instruction

The program counter does not store the current instruction.

  • PC stores the next instruction address.
  • Instruction register stores the current instruction.

Fix: Link the PC to the word next in your answer.

4 marks Explain

Question 1

Explain the role of the program counter, MAR, MDR, and instruction register during the fetch stage.

Show answer guide

A strong answer explains what each register holds.

  • The program counter stores the address of the next instruction.
  • That address is copied into the MAR so memory can be accessed.
  • The instruction is fetched from memory through the MDR.
  • The instruction is then placed in the instruction register ready for decoding.
6 marks Explain and apply

Question 2

A CPU is executing an addition instruction. Explain how the instruction cycle allows that instruction to be fetched, decoded, and executed, and include at least one implication if the next instruction address were not updated correctly.

Show answer guide

The best answers follow the sequence and then explain the consequence.

  • The CPU fetches the instruction using the PC, MAR, MDR, and instruction register.
  • The control unit decodes the opcode to identify that an addition is required.
  • The execute stage carries out the addition, often using the ALU and registers such as the accumulator.
  • The program counter is then ready to point to the next instruction unless a jump changes it.
  • If the next address were wrong, the CPU could fetch the wrong instruction or repeat the same one.

Glossary

Quick glossary

Instruction cycle
The repeated fetch, decode, and execute process used by the CPU.
Fetch
The stage where the next instruction is brought from memory into the CPU.
Decode
The stage where the control unit interprets the instruction.
Execute
The stage where the instruction is carried out.

Remember

Fetch needs the right register story

  • PC means next instruction address.
  • MAR means address to access now.
  • MDR means data or instruction in transit.
