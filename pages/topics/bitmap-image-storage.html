Overview

What you should know by the end of this lesson

A bitmap image stores a picture as a grid of tiny picture elements called pixels. Each pixel needs binary data to describe its colour, so image quality and storage size are closely linked.

Learning goals

  • Explain how bitmap images are stored.
  • Define a pixel clearly.
  • Link pixel data to image size and quality.
  • Compare bitmap strengths with bitmap limitations.

Quick memory hooks

  • Bitmap: grid of pixels.
  • Pixel: one picture element.
  • More pixels: more detail.
  • More data: larger file.

Pixels

Bitmap images are built from rows and columns of pixels

A bitmap stores the image as a rectangular grid. Each position in the grid represents one pixel, and each pixel stores colour information. When many pixels are shown together, the full picture appears.

Why bitmaps can look blocky

If a bitmap is enlarged too far, each pixel becomes more visible, so edges can appear jagged or blocky. This is one of the main trade-offs of pixel-based images.

Colour

Each pixel needs binary data to describe its colour

The system stores a binary value for each pixel. More bits per pixel mean more possible colours, but also more storage required for the image.

Monochrome idea A very simple image may need only 1 bit per pixel for black or white.
More colours Higher colour precision needs more bits for every pixel.
Direct effect If every pixel uses more bits, the full file size increases.
Display quality Richer colour data can make gradients and photographs look smoother.

Storage

File size grows as the image stores more pixel information

The more pixels an image contains, and the more bits used for each pixel, the more data the bitmap must store. That is why detailed pictures can take up large amounts of space.

Simple idea for estimation

If an image has 100 x 100 pixels, it contains 10,000 pixels in total. If each pixel uses 8 bits, the raw pixel data needs 80,000 bits before any extra file information or compression is considered.

Trade-Offs

Bitmaps are strong for detailed images but storage-heavy at larger sizes

Exam Traps

Common mistakes with bitmap images

Confusing bitmap with the display screen

A bitmap is the stored image representation, not the whole monitor itself.

Fix: Talk about pixels stored in the image file or memory.

Ignoring the link between quality and size

Better image detail usually means more stored data.

Fix: Link more pixels or more colour data to a larger file.

Forgetting that zooming can reveal pixels

Bitmap images do not scale perfectly forever.

Fix: Mention blockiness or jagged edges when enlarged.

Glossary

Quick glossary

Bitmap
An image stored as a grid of pixels.
Pixel
One picture element in the image.
Raster
Another term for a pixel-grid image structure.
Colour data
The stored value that tells the system what colour to show.

Remember

Every pixel costs data

  • Bitmaps store pictures as pixel grids.
  • More pixels and richer colour mean more storage.
  • Scaling up too far can reveal the pixel structure.
