Overview

What you should know by the end of this lesson

Binary arithmetic follows familiar arithmetic ideas, but the rules only use the digits 0 and 1. Strong exam answers show the working clearly, especially where carries, borrows, or BCD corrections appear.

Learning goals

  • Apply the basic rules of binary addition.
  • Use borrowing to subtract in binary.
  • Recognise simple binary multiplication and division.
  • Explain when BCD arithmetic needs correction.

Quick rules

  • 1 + 1 = 10 in binary.
  • 1 + 1 + 1 = 11 in binary.
  • Borrow: take 1 from the next column.
  • BCD: each nibble must stay 0000 to 1001.

Addition

Binary addition uses carrying in the same way as denary

Start from the right and work left. The main rule to remember is that 1 + 1 produces 10, so you write 0 and carry 1.

0 + 0 = 0 0 + 1 = 1 1 + 1 = 10 1 + 1 + 1 = 11

Worked example

Add 1011 + 0110. From right to left: 1 + 0 = 1, 1 + 1 = 10 so write 0 carry 1, 0 + 1 + carry 1 = 10 so write 0 carry 1, 1 + 0 + carry 1 = 10. Final answer: 10001.

Subtraction

Binary subtraction often needs borrowing from the next column

Subtract from right to left. If you need to take 1 from 0, borrow from the next column to the left, just as you would in denary subtraction.

0 - 0 Gives 0.
1 - 0 Gives 1.
1 - 1 Gives 0.
0 - 1 Borrow from the next column first.

Worked example

Subtract 1000 - 0011. Borrow from the left so the two rightmost columns can be completed. The result is 0101, which is 5 in denary.

Multiply And Divide

Binary multiplication and division often reduce to shifting and repeated subtraction

Multiplying by 10 in binary shifts the number left by one place, which doubles it. Dividing by 10 shifts right by one place, which halves it for whole-number cases.

Operation Binary example Meaning
Multiply by 2 1011 x 10 = 10110 Shift left once
Divide by 2 10110 / 10 = 1011 Shift right once
Multiply by 3 101 x 11 Add a shifted copy as part of the working

BCD

BCD arithmetic must keep each 4-bit digit inside the valid range

In BCD, valid digits run from 0000 to 1001. If an addition produces an invalid digit such as 1010 to 1111, the result needs decimal correction.

Simple correction example

Add decimal 5 and 7 in BCD: 0101 + 0111 = 1100. The nibble 1100 is not a valid BCD digit, so add 0110 to correct it. That gives 1 0010, which represents decimal 12.

Why BCD needs this extra step

Pure binary arithmetic is happy with any 4-bit result from 0000 to 1111, but BCD only allows the patterns for decimal 0 to 9 in each digit position.

Exam Traps

Common mistakes with binary arithmetic

Forgetting the carry

Binary addition breaks down if the carry is not moved into the next column.

Fix: Write carries above the columns if the working is getting busy.

Borrowing from the wrong place

In subtraction, the borrow must come from the next higher column to the left.

Fix: Mark the changed columns clearly.

Accepting an invalid BCD nibble

A result like 1100 is fine in binary but invalid as one BCD digit.

Fix: Check each BCD digit stays between 0000 and 1001.

Question 2

Explain why BCD arithmetic sometimes needs a correction step even when normal binary arithmetic appears to be complete.

Show answer guide
  • Each BCD digit is stored in a 4-bit group representing decimal 0 to 9 only.
  • Normal binary addition can produce values from 1010 to 1111.
  • Those patterns are invalid for a single BCD digit.
  • A correction step is used so the digit becomes a valid decimal representation again.

Glossary

Quick glossary

Carry
An extra value moved into the next column during addition.
Borrow
A value taken from the next higher column in subtraction.
BCD correction
The extra step needed when a BCD digit becomes invalid.
Shift
Moving binary digits left or right by place value.

Remember

  • Carrying and borrowing should be visible on the page.
  • Left shift doubles and right shift halves whole numbers.
  • BCD digits cannot go beyond 1001 without correction.
