Overview

What you should know by the end of this lesson

Emulation is about recreating the behaviour of one computer system on another. In Unit 2, strong answers explain what is being imitated, why emulation can be useful, and why emulated systems may not perform the same as native ones.

Learning goals

  • Define emulation clearly.
  • Explain common uses of emulation.
  • Describe the main benefits and drawbacks of emulation.
  • Recommend when emulation is a suitable choice for a scenario.

Quick memory hooks

  • Emulation: one system imitates another system's behaviour.
  • Use: compatibility, legacy software, and preservation.
  • Benefit: old or different software can run without the original hardware.
  • Trade-off: emulation can be slower or less exact than native operation.

Where it appears

This topic appears when the exam asks about legacy systems, compatibility between platforms, testing, training, or the impact of choosing one architecture approach over another.

Definition

Emulation recreates the behaviour of another system

An emulator is designed to imitate how another computer system, processor, console, or platform behaves. That lets software built for the original system run on different hardware or in a new environment.

Original system has its own hardware or architecture Emulator reproduces that behaviour Host system runs the emulated environment

Simple example

A modern computer may run an emulator so that a program written for an older or different platform can still work. The host machine is not magically becoming the original hardware. It is imitating enough of its behaviour for the software to run.

Host system The real machine that is running the emulator.
Target system The original machine or platform being imitated.
Compatibility Software can keep working even when the original hardware is unavailable.
Overhead Recreating another system's behaviour takes extra work.

Uses

Emulation is useful when the original platform is unavailable or impractical

Emulation is often chosen when software, training material, or system behaviour still matters but the original hardware is unavailable, expensive, outdated, or difficult to maintain.

Legacy software

Organisations may still need old software that was built for a previous platform or operating environment.

Preservation

Emulation can help preserve older systems, games, or specialist applications after original hardware becomes rare.

Testing and education

Students or developers can explore how a system behaves without needing the real target hardware.

Why this matters in exams

Exam questions usually want more than "to run old software". Strong answers explain the reason the original platform is a problem and why imitation is a practical solution.

Impact

Emulation improves access and compatibility, but it can reduce performance

Emulation has real benefits, but it also adds overhead because the host system must reproduce another platform's behaviour. That can affect speed, precision, or the range of features that work perfectly.

Impact area Positive effect Possible limitation
Compatibility Old or different software can run on newer hardware. Not every feature or hardware behaviour may be recreated perfectly.
Cost and access A user may avoid buying or maintaining the original system. Setup can still be complex, especially for specialist platforms.
Performance Software becomes available where it otherwise could not run. Emulation can be slower than native execution because the host is translating or reproducing behaviour.

Key exam phrase

If a question asks about the impact of emulation, useful phrases include compatibility, legacy support, performance overhead, and recreating behaviour.

Comparison

Choose emulation when imitation is more important than native speed

Good answers explain when emulation is worth it and when it is not. The strongest justification matches the need for compatibility to the cost of extra overhead.

Situation Why emulation may suit it Why it may not
Running legacy software The original platform may no longer be practical to keep. Performance or hardware-specific features may not match the original perfectly.
Training or demonstration Learners can access the system behaviour without physical target hardware. It may not fully represent every real-world detail of the original setup.
Maximum performance production work Only suitable if compatibility matters more than speed. Native execution is often better if the right platform is available.

Scenario thinking

If the exam scenario focuses on keeping older software usable, preserving a system, or accessing another platform without the original hardware, emulation is often a strong answer. If the scenario focuses on speed alone, emulation may be a weaker choice.

Exam Traps

Common mistakes students make with emulation

Emulation questions often lose marks when answers say it is just "making software work" without explaining what is being imitated or what trade-off comes with that choice.

Defining emulation too loosely

Saying only that it "runs old programs" misses the main idea.

  • Emulation imitates another system's behaviour.
  • The host system is recreating what the target system would do.

Fix: Use the phrase imitates another system.

Assuming emulation is always as fast as native use

Recreating another platform usually adds overhead.

  • Compatibility can improve.
  • Performance can still fall.

Fix: Mention the trade-off between access and speed.

Ignoring why the original platform is unavailable

The best scenario answers explain why emulation is needed in the first place.

  • Old hardware may be rare or costly to maintain.
  • A different architecture may be the only system currently available.

Fix: Link emulation to compatibility or preservation.

4 marks Explain

Question 1

Explain what emulation is and give one reason why a business or college might use it.

Show answer guide

A strong answer defines the imitation clearly and then links it to a realistic use case.

  • Emulation means one system imitates the behaviour of another system or platform.
  • This allows software written for the original platform to run on the host system.
  • One reason to use it is to keep legacy software usable when the original hardware is old, expensive, or unavailable.
6 marks Discuss and justify

Question 2

A college needs access to an old specialist application that was written for a platform it no longer owns. Discuss whether emulation is a suitable solution and justify your answer.

Show answer guide

The best answers weigh the benefit of access against the cost of overhead or imperfect recreation.

  • Emulation may be suitable because it can recreate the original platform on newer hardware.
  • That means the old software may still be usable without buying and maintaining the original system.
  • This can support compatibility, preservation, and training.
  • However, emulation may be slower than native use or may not reproduce every feature perfectly.
  • A strong conclusion states that emulation is suitable if access to the old software matters more than maximum performance.

Glossary

Target system
The original machine or platform being imitated.
Legacy software
Older software that may still be needed even though its original platform is outdated.

Remember
