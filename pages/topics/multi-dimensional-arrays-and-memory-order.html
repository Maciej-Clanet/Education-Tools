Multi-dimensional arrays and memory order

Overview

What you should know by the end of this lesson

Arrays do not stop at a single line of values. They can organise data in tables and larger structures too. Unit 2 also expects you to understand that the logical shape of a multi-dimensional array is not the same thing as the order used to place its values into memory.

Learning goals

  • Define one-dimensional, two-dimensional, and multi-dimensional arrays.
  • Explain how multiple indexes locate one value.
  • Describe row-major and column-major order accurately.
  • Link memory order to how values are laid out in storage.

Quick memory hooks

  • 1D: one index, one line of values.
  • 2D: two indexes, rows and columns.
  • Multi-dimensional: more than two indexes.
  • Memory order: the sequence values follow in storage.

Where they appear

These structures appear in image data, room layouts, seating plans, timetables, game worlds, and any system that needs several categories of position at once.

Dimensions

Multi-dimensional arrays extend the same indexed idea

A one-dimensional array uses one position number. A two-dimensional array uses two indexes, usually treated as row and column. A multi-dimensional array uses more than two indexes, so each extra dimension gives the program another layer of structure.

1D scores[4] 2D seats[2][5] 3D timetable[week][day][period]
Why it works Each index narrows the search until one exact value is chosen.
Typical software use 2D arrays suit tables and images, while 3D arrays can model layers such as time, depth, or locations.
Hardware link The program still needs a consistent rule to convert multiple indexes into stored addresses.
Key implication More dimensions make the structure richer, but access rules must still stay precise.

Worked example

A school timetable could be stored as timetable[week][day][period]. That means one value is chosen by giving three pieces of position information instead of one.

If timetable[1][3][2] is "Maths", the program has reached one exact lesson by moving through week 2, day 4, and period 3 in zero-based indexing.

Array type Indexes used Best mental model Example use
One-dimensional 1 A line or list of values Exam scores for one class
Two-dimensional 2 A grid with rows and columns Seat occupancy or image pixels
Multi-dimensional 3 or more A grid with extra layers of position Timetables, 3D game space, or sensor history

Storage Order

Memory order decides how the values are laid out in storage

A two-dimensional array may look like a neat grid on paper, but memory is still organised as a sequence of locations. Row-major and column-major order are two rules for deciding which values are placed next to each other in that sequence.

Row-major store a full row, then the next row Column-major store a full column, then the next column Layout same grid, different storage sequence

Logical view

The array still represents the same rows and columns either way.

Memory view

The stored order changes which values sit next to each other in memory.

Why it matters

Address calculation and access efficiency depend on the chosen order.

Worked layout example

Take this 2 x 3 array:

[ [11, 12, 13],
  [21, 22, 23] ]

In row-major order, memory stores: 11, 12, 13, 21, 22, 23

In column-major order, memory stores: 11, 21, 12, 22, 13, 23

The logical array has not changed. Only the rule used to place its values into memory has changed.

    Row-major versus column-major

    Feature Row-major order Column-major order
    Main sequence Finish one row before starting the next Finish one column before starting the next
    Values stored together Horizontal neighbours Vertical neighbours
    Best exam wording Stored row by row Stored column by column
    Main caution Do not confuse with the on-page grid itself Do not assume it changes the meaning of the data

    Comparison

    Separate the shape of the array from the order in memory

    One of the most useful revision habits here is to split the topic into two questions: how many indexes identify the value, and in what order are those values stored in memory?

    Idea What it describes Example wording Why it matters
    Dimensions The number of indexes needed 1D, 2D, or 3D array Shows the structure of the data.
    Indexes The chosen position inside that structure grid[2][4] Lets the program retrieve one exact value.
    Memory order The sequence used in storage Row-major or column-major Changes adjacency in memory and address rules.
    Logical meaning What the data represents in the real problem Seat map, timetable, or image Keeps the explanation tied to the scenario.

    Talk about dimensions when...

    The question asks how the data is organised or how many categories of position are needed.

    Talk about memory order when...

    The question asks how the values are laid out internally or which elements are stored next to each other.

    Exam reminder

    A 2D array can use either row-major or column-major order, so the number of dimensions and the memory order are not the same thing.

    Exam Traps

    Common mistakes students make with multi-dimensional arrays

    These questions often catch learners who know the vocabulary but blur together structure, indexing, and storage order.

    Confusing dimensions with memory order

    Some answers say that a 2D array is row-major by definition, which is not correct.

    • Dimensions describe the shape of the array.
    • Memory order describes how that shape is stored.

    Fix: Explain the structure first, then the storage rule.

    Losing track of zero-based indexing

    Students sometimes name the correct row or column but shift the answer by one because they forget index 0.

    • [0][0] is the first position, not the second.
    • More dimensions mean more chances to slip by one.

    Fix: If the question uses programming notation, count from 0 unless told otherwise.

    Assuming storage order changes the logical grid

    Row-major and column-major order change layout in memory, not what the array represents to the program.

    • The same element still exists in the same logical position.
    • The difference is which values sit next to it in memory.

    Fix: Use the phrase "stored in memory" when you explain row-major or column-major order.

    Question 1

    Explain why a three-dimensional array would be suitable for storing room bookings for different weeks, days, and lesson periods in a college timetable system.

    Show answer guide

    A strong answer links each dimension to a real category in the scenario.

    • The booking system needs separate positions for week, day, and lesson period.
    • A three-dimensional array provides three indexes to represent those categories.
    • The program can retrieve one booking directly by supplying all three indexes.
    • This keeps the structure organised and matches the timetable layout logically.
    5 marks Describe and compare

    Question 2

    A 2 x 3 array stores values [ [4, 5, 6], [7, 8, 9] ]. Describe the order of values in row-major memory and in column-major memory, then explain why the logical grid is still the same in both cases.

    Show answer guide

    The best answers give both sequences and then explain the difference between storage layout and logical meaning.

    • Row-major order stores: 4, 5, 6, 7, 8, 9.
    • Column-major order stores: 4, 7, 5, 8, 6, 9.
    • Row-major stores one full row before the next row.
    • Column-major stores one full column before the next column.
    • The array still represents the same 2 x 3 grid logically, even though the memory sequence differs.

    Glossary

    Quick glossary

    Multi-dimensional array
    An array that uses more than two indexes to locate a value.
    Row-major order
    A storage rule that places values into memory row by row.
    Column-major order
    A storage rule that places values into memory column by column.
    Index
    The position number used to identify a value inside an array.

    Remember

    Ask two separate questions

    • How many indexes are needed to locate the value?
    • How are the values stored in memory?
    • Those two answers are related, but they are not identical.
