Stacks and queues

Learn how two simple data structures control the order in which data is stored and removed. This lesson focuses on the LIFO rule used by stacks, the FIFO rule used by queues, and when each one makes more sense in software and hardware.

Overview

What you should know by the end of this lesson

Stacks and queues are both ordered collections of data, but they remove items in different ways. Exam questions often test whether you can name the rule, choose the right structure for a scenario, and explain why that choice works.

Learning goals

  • Define a stack and explain the LIFO rule.
  • Define a queue and explain the FIFO rule.
  • Name the common operations used with each structure.
  • Choose the better structure for a given scenario.

Quick memory hooks

  • Stack: add to the top, remove from the top.
  • Queue: join at the back, leave from the front.
  • LIFO: the newest item leaves first.
  • FIFO: the oldest item leaves first.

Where they appear

Stacks are common in function calls, undo features, and browser history. Queues are common in printer spooling, task scheduling, keyboard buffers, and packet handling.

Stack

A stack stores data using the LIFO rule

LIFO means last in, first out. The most recently added item is the first one to be removed. You can picture a stack like a pile of trays: you add to the top, and you remove from the top.

Push add an item to the top Pop remove the top item Peek read the top item without removing it
Why it works A stack is useful when the newest thing must be handled first.
Typical software use Undo systems return through the most recent actions first.
Typical hardware use Function calls and interrupts often rely on stack-style memory.
Key implication You do not normally remove items from the middle of a stack.

Stack diagram

Both actions happen at the same end. New items are pushed onto the top, and the same top item is the first one popped off.

Push -> top The newest item joins the top of the stack.
Pop <- top The same top item is the first one removed.

Only the top is active, so older items stay underneath until the newer ones have gone.

Push Task D -> top Pop Task C <- top
Top
Task C Leaves first
Task B Waits below
Task A Oldest item

The newest item sits closest to leaving.

Worked example

If a stack stores Task A, then Task B, then Task C, the next pop returns Task C. The newest value leaves first.

Queue

A queue stores data using the FIFO rule

FIFO means first in, first out. The earliest item added is the first one to leave. A queue works like people lining up: new items join at the back, but service happens at the front.

Enqueue add an item to the back Dequeue remove the front item Front inspect the next item to leave

Fair processing

Queues are useful when items should be handled in the order they arrive.

Typical software use

Print jobs are usually processed in the same order they were requested.

Typical hardware use

Input buffers and packet buffers often keep arrival order rather than newest-first order.

Queue diagram

A queue uses opposite ends. The front item leaves first while a new item joins at the back, so arrival order stays intact.

Front: dequeue here Back: enqueue here
Packet 1 Next out
Packet 2 Waiting
Packet 3 Waiting
Packet 4 Newest in line
Packet 1 leaves Packet 5 joins

The loop above shows the pattern repeating: oldest item out at the front, new item in at the back.

Worked example

If a queue stores Packet 1, then Packet 2, then Packet 3, the next dequeue returns Packet 1. The oldest waiting item leaves first.

Comparison

Choose the right structure for the job

The main difference is not what the structure stores. It is the rule that controls removal. That rule affects fairness, speed of access, and whether the newest or oldest item matters most.

Feature Stack Queue
Removal rule Last in, first out First in, first out
Main operations Push, pop, peek Enqueue, dequeue, front
Best when The newest item should be handled first Items should be handled in arrival order
Common examples Undo, call stack, browser history Printer jobs, buffers, task scheduling
Main caution Older items can stay buried for longer New urgent items may have to wait their turn

Use a stack when...

Reversal matters, nested actions occur, or the most recent step must be undone or completed first.

Use a queue when...

Fairness matters, arrival order should be preserved, or work should be handled one-by-one in the order it appears.

Exam reminder

If a question asks what comes out next, always focus on the order rule before naming the answer.

Exam Traps

Common mistakes students make with stacks and queues

Many wrong answers happen because the names sound familiar but the order rule gets lost. These are the mistakes exam questions often try to expose.

Mixing up LIFO and FIFO

Students sometimes remember that both structures are ordered, but forget which one removes the newest item and which one removes the oldest.

  • A stack removes from the top.
  • A queue removes from the front.

Fix: Ask yourself, "Does the newest item go first or does waiting order matter?"

Using the wrong operation names

A question may be testing vocabulary as well as understanding. Saying pop for a queue or enqueue for a stack can lose marks.

  • Stack: push, pop, peek
  • Queue: enqueue, dequeue, front

Fix: Link the operation to the direction of movement, not just the data structure name.

Choosing by topic instead of behaviour

Students sometimes say "printer means stack" or "browser means queue" because they remember the example, not the reason.

  • Undo and nested calls favour the newest item first.
  • Printing and buffering usually preserve arrival order.

Fix: In every scenario, explain the order rule before naming the structure.

      3 marks Explain

      Question 1

      Explain why a browser back history is usually modelled as a stack rather than a queue.

      Show answer guide

      A strong answer makes the order rule explicit and then links it to the example.

      • The most recent page visited is the first one the user wants to return to.
      • A stack uses last in, first out, so the newest page is removed first.
      • A queue would return the oldest page first, which does not match browser back behaviour.
      4 marks Describe and justify

      Question 2

      A school printer receives jobs from Mia, Sam, Priya, and Leo in that order. Describe how a queue would process the jobs and explain why a stack would be less suitable.

      Show answer guide

      The best answers describe the order clearly and compare it with the wrong structure.

      • The queue stores jobs in arrival order.
      • Mia's job would be processed first, then Sam's, then Priya's, then Leo's.
      • A queue uses first in, first out, which is fair for waiting print jobs.
      • A stack would process Leo first, which would ignore arrival order and be less fair.

      Glossary

      Quick glossary

      LIFO
      Last in, first out. The newest item leaves first.
      FIFO
      First in, first out. The oldest item leaves first.
      Push and pop
      Add to the top of a stack, then remove from the top.
      Enqueue and dequeue
      Add to the back of a queue, then remove from the front.

      Remember

      One rule changes everything

      • Stacks care most about the newest item.
      • Queues care most about arrival order.
      • Exam scenarios usually reveal the answer through that rule.
