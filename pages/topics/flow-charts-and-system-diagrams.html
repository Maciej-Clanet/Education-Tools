Overview

What you should know by the end of this lesson

Flow charts and system diagrams both help represent how a system works, but they do different jobs. Flow charts focus on the steps in a process, while system diagrams show the components and the movement of data between them.

Learning goals

  • Recognise common flow chart symbols.
  • Explain what a system diagram shows.
  • Choose the best diagram for a problem.
  • Interpret data flow between components.

Quick memory hooks

  • Flow chart: steps in order.
  • Decision: branch based on a condition.
  • System diagram: components and links.
  • Arrow: direction of flow matters.

Why this matters

In the exam, diagrams are often used to explain a process, justify a design, or show how data moves through a system. Clear modelling helps you show understanding quickly.

Flow Charts

Flow charts show the order of steps in a process

A flow chart breaks a process into a sequence of actions. It is especially useful when you want to show decisions, loops, and the order in which operations happen.

Start/End where the process begins or stops Process an action or calculation Decision choose between paths

Common symbol meanings

Symbol type Meaning Typical use
Terminator Start or end point Beginning or finish of a process
Process An action or task Calculate total, save file, update record
Input/Output Data entering or leaving the system Read sensor, display message, print receipt
Decision A yes/no test Is password correct? Is stock available?
Flow line Direction of the process Connects one step to the next

Reading a simple process

A login flow chart might start with the user entering a password, then move to a decision: “Is the password correct?” If yes, access is granted. If no, an error message is shown and the process may loop back for another attempt.

System Diagrams

System diagrams show components and the movement of data

A system diagram is less about step-by-step logic and more about the structure of the system. It shows what parts are involved and how data passes between them.

Input Where data enters, such as a sensor or keyboard.
Process The component or software that handles the data.
Storage Where data is held for later use or retrieval.
Output Where the final result is sent or shown.

Example system model

In a barcode checkout system, the scanner is the input, the till software is the process, the product database is storage, and the display or receipt printer is the output. Arrows show the path the data follows between those parts.

Why arrows matter

The direction of the arrow tells you where the data is going. A reversed arrow can change the meaning completely, so diagrams should show data flow carefully and consistently.

Choosing and Applying

Choose the right diagram for the question you need to answer

The best diagram depends on what you need to explain. If the exam asks how a process moves through decisions and steps, a flow chart is often best. If it asks how data moves between hardware, software, or storage, a system diagram is usually stronger.

Flow chart or system diagram?

Question focus Best diagram Reason
Show the steps in a password check Flow chart It models process order and decisions clearly
Show how data moves through a booking system System diagram It highlights components and data paths
Explain where a process loops back Flow chart Loops are easy to show with flow lines
Explain how input, process, storage, and output connect System diagram It gives a structural overview of the system

How to build a strong diagram answer

  • Start by identifying the purpose of the diagram.
  • Use standard symbols or clearly labelled components.
  • Show direction with arrows.
  • Keep the labels short and accurate.
  • Make sure the diagram matches the scenario in the question.

Exam Traps

Common mistakes with flow charts and system diagrams

Students usually lose marks when they choose the wrong diagram or use arrows and labels too loosely.

Using the wrong diagram type

A system diagram cannot replace a step-by-step process model, and a flow chart does not show system structure as clearly.

Fix: Decide whether the question is about process order or system structure first.

Missing or unclear arrows

Without arrow direction, the examiner cannot tell how the data or control moves.

Fix: Add arrows deliberately and check each one still makes sense when read aloud.

Non-standard symbols

Inventing shapes or leaving them unlabeled makes the diagram harder to interpret.

Fix: Use the standard symbol meanings and add short labels where needed.

4 marks Explain

Choosing the right model

Explain why a flow chart would be more useful than a system diagram for showing the steps in an online password-check process.

Answer guidance

A strong answer should explain that a flow chart shows the order of actions and decisions, such as entering the password, checking it, branching to success or failure, and looping back if needed.

  • State that the question is about process steps.
  • Explain that a flow chart shows sequence clearly.
  • Mention decisions and branching.
  • Contrast this with the structural focus of a system diagram.
6 marks Apply

Modelling data flow

A temperature sensor sends readings to a control program. The program stores the readings and sends an alert to a display if the temperature is too high. Explain how a system diagram could represent this.

Answer guidance

A strong answer should identify the sensor as the input, the control program as the processing stage, storage for saved readings, and the display as the output. Arrows should show the reading moving from the sensor to the program, then to storage and to the display when the alert condition is met.

  • Identify input, process, storage, and output.
  • Explain the direction of the arrows.
  • Show that readings are stored as well as checked.
  • Link the alert output to the high-temperature condition.
