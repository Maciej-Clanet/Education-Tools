Overview

What you should know by the end of this lesson

When data is spread across multiple systems, organisations can gain flexibility and shared access, but they also take on new costs, risks, and management challenges.

Learning goals

  • Explain why organisations use more than one system.
  • Discuss the impact on access, cost, and productivity.
  • Evaluate implementation and security concerns.
  • Reach balanced conclusions in scenario questions.

Quick memory hooks

  • Access: who can reach the data?
  • Cost: what does setup and upkeep require?
  • Productivity: does sharing help work flow?
  • Security: does wider access increase risk?

Typical situations

Schools, businesses, hospitals, warehouses, and cloud-based teams all rely on more than one system when they need shared data, remote access, or coordinated services.

Purpose

Organisations use multiple systems to share work, services, and information

A single computer may not be enough when many users need the same data or when different departments, sites, or devices must work together.

Shared accessMany users can reach the same records or files without duplicate copies everywhere.
Specialised rolesDifferent systems can focus on storage, processing, reporting, or user access.
Remote workUsers in different locations can still use the same data set.
ScalabilityMore systems can be added as demand grows.

Implications

Using data across multiple systems creates both benefits and risks

Unit 2 expects you to discuss the impact, not just describe the setup. The same design choice can improve one area while making another harder to manage.

Factor Potential benefit Potential issue
Access Users can reach data from different places or devices Access control becomes more important
Cost Resources can be shared efficiently Setup, licensing, infrastructure, and support can be expensive
Implementation Systems can be tailored to different roles Migration, integration, and testing can be complex
Productivity Staff can collaborate and avoid re-entering data Poor integration can slow work down instead
Security Central controls may improve oversight More systems and connections create more attack surfaces

Judgement

The best answers weigh the overall impact rather than assuming distributed data is always better

A strong judgement links the system design to the organisation's needs, staff, budget, and risk level.

Benefit easier collaboration Benefit remote access Risk more security exposure

Worked example

A school may store student data in a central management system that teachers access from many classrooms. This improves access and productivity, but it also means the school must manage user permissions carefully and protect the network and devices that connect to the data.

Exam Traps

Common mistakes with data across multiple systems

These questions usually lose marks when answers only describe the convenience and ignore the risks or implementation effort.

Assuming more access is automatically good

Convenience can also increase security risk if controls are weak.

Fix: Balance access against permissions and protection.

Ignoring implementation cost

Connecting systems takes time, testing, migration, and support.

Fix: Mention setup and maintenance, not just the final result.

Only listing factors

Marks often depend on applying them to the scenario.

Fix: Explain how each factor changes the organisation’s situation.

Question 1

Explain how using data across multiple systems can improve productivity in an organisation.

Show answer guide
  • Shared data can be accessed by more than one user or department.
  • This reduces repeated entry of the same information.
  • Staff can work from consistent up-to-date records.
  • Collaboration can improve because everyone works from the same data.
  • Time is saved, which can improve productivity.
6 marksDiscuss

Question 2

A business wants staff in several branches to access shared customer data. Discuss the likely benefits and concerns of this approach.

Show answer guide
  • Shared access can improve consistency and customer service.
  • Staff in different branches can use the same up-to-date records.
  • Productivity can improve because data does not need to be re-entered.
  • Implementation may require integration, testing, and staff training.
  • Security becomes more important because more systems and users can reach the data.
  • A balanced judgement should weigh the gains against cost and risk.

Glossary

Quick glossary

Implementation
The process of putting a system into use successfully.
Integration
Making separate systems work together.
Access control
Rules about who can reach or change data.
Productivity
How effectively users can complete their work.

Remember

Weigh both sides

  • Shared systems can improve access and productivity.
  • They can also increase cost, complexity, and security risk.
  • Strong answers balance benefits against concerns.
