Overview

What you should know by the end of this lesson

Digital data units tell us how much information is being stored, processed, or transmitted. Exam questions often become easier as soon as you spot whether the value is measured in bits or bytes.

Learning goals

  • Define bit, byte, kilobyte, megabyte, and gigabyte.
  • Convert between common data units accurately.
  • Explain the difference between size and speed.
  • Use unit language clearly in exam answers.

Quick memory hooks

  • Bit: one binary digit, 0 or 1.
  • Byte: 8 bits.
  • Capacity: how much data fits.
  • Rate: how fast data moves.

Why it matters

Hardware specifications, file sizes, memory capacity, and network speeds all rely on these units. If you confuse megabits with megabytes, your final answer can be eight times too large or too small.

Foundations

Bits and bytes are the core building blocks of digital data

A bit is the smallest unit of data and can hold one of two values: 0 or 1. A byte groups eight bits together, which is why bytes are commonly used for storage sizes.

1 bit one binary choice 1 byte 8 bits Uppercase B bytes Lowercase b bits
Bit Used when talking about binary states, bandwidth, and raw signal information.
Byte Commonly used for file size, storage capacity, and memory size.
Example A 24-byte record contains 192 bits because 24 x 8 = 192.
Exam clue Watch the symbol carefully: Mb is not the same as MB.

Multiples

Larger units help us describe useful amounts of data

Once data gets larger than a few bytes, bigger units make values easier to read. In many computing exam questions, sizes are treated as powers of 1024 unless the context clearly says otherwise.

Unit Common meaning Approximate size
Kilobyte (KB) Small file or memory block 1024 bytes
Megabyte (MB) Image, song, or program file 1024 KB
Gigabyte (GB) Large storage amount 1024 MB
Terabyte (TB) Very large drive or backup store 1024 GB

Worked conversion

If a memory stick holds 4 GB, that is 4 x 1024 = 4096 MB. If you want the answer in bytes, continue multiplying by 1024 at each step.

Interpretation

Storage capacity and transfer rate use similar units for different ideas

Capacity tells you how much data can be stored. Transfer rate tells you how quickly data moves. The same prefixes appear in both contexts, but network speeds are often written in bits per second rather than bytes.

Storage example A file size of 80 MB describes how much space the file uses.
Transfer example A connection speed of 80 Mb/s describes how much data can move each second.
Useful conversion Divide by 8 to change bits into bytes, so 80 Mb/s = 10 MB/s.
Exam habit Write the unit beside every answer so the examiner can see exactly what you calculated.

Why the distinction matters

If a 200 MB video is downloaded over a 40 Mb/s link, you must convert the rate to megabytes per second before estimating time. Otherwise you will compare unlike units and misjudge the download speed.

Exam Traps

Common mistakes with digital data units

These marks are often lost through careless unit handling rather than difficult theory.

Mixing up bits and bytes

Many students read the prefix and miss the final letter. Mb and MB are not interchangeable.

Fix: Underline the unit first, then convert before you calculate.

Forgetting the factor of 8

Network rates are often measured in bits per second, but file sizes are usually measured in bytes.

Fix: Use 1 byte = 8 bits before comparing values.

Leaving the answer unitless

A number on its own is incomplete in data-size questions.

Fix: Always include the final unit, such as MB, Gb, or seconds.

Question 1

Explain the difference between a bit and a byte, and give one reason why both terms are still needed in computing.

Show answer guide
  • A bit stores one binary value: 0 or 1.
  • A byte contains 8 bits.
  • Bits are often used for bandwidth or very small binary states.
  • Bytes are commonly used for file size, RAM, and storage capacity.
5 marks Calculate

Question 2

A file is 240 MB and a network runs at 80 Mb/s. Explain how to estimate the transfer time and why unit conversion is necessary.

Show answer guide
  • The file size is in megabytes but the rate is in megabits per second.
  • Convert the rate to megabytes per second by dividing by 8, so 80 Mb/s = 10 MB/s.
  • Divide file size by rate: 240 MB / 10 MB/s = 24 seconds.
  • Unit conversion is needed because bits and bytes are different units.

Glossary

Quick glossary

Bit
The smallest binary unit, holding 0 or 1.
Byte
A group of 8 bits.
Capacity
The amount of data that can be stored.
Bandwidth
The amount of data that can move per second.

Remember
