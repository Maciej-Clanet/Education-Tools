Overview

What you should know by the end of this lesson

Software questions often test whether you understand what the software is for, how it supports the user or system, and what factors should shape the final choice.

Learning goals

  • Explain the purpose of utility software.
  • Explain the purpose of application software.
  • Compare proprietary and open source approaches.
  • Judge software choice using suitability and implications.

Quick memory hooks

  • Utility: maintains or protects the system.
  • Application: helps the user do a task.
  • Open source: source code is available.
  • Choice: think about needs and risk.

Big picture

Utility software keeps systems usable and secure. Application software helps users produce work or media. Open source adds further questions about cost, control, support, and customisation.

Utilities

Utility software supports system maintenance, protection, and management

Utility software is designed to keep the computer system healthy, safe, efficient, or manageable rather than to produce the user's main work directly.

Anti-malware tools Help detect or block malicious software and reduce security risk.
Backup tools Support data protection and recovery planning.
Disk tools Help check, organise, or clean storage so the system remains usable.
Compression utilities Reduce file size or package data for storage and transfer.

Applications

Application software helps users complete specific tasks

Application software exists for the user’s main activity, such as writing, designing, coding, analysing data, or communicating.

Type Purpose Example use Important choice factors
Office software Create documents, spreadsheets, and presentations School or business admin work Compatibility, ease of use, file sharing
Media software Edit images, audio, or video Creative production tasks Performance, features, hardware support
Communication software Message, meet, or collaborate Remote teamwork Security, reliability, accessibility
Specialist software Support a specific profession or process Accounting, CAD, diagnosis Task fit, training, support

Open Source

Open source software can offer flexibility and lower cost, but it still needs careful judgement

Open source software makes source code available, which can allow inspection, modification, and community improvement. That can be a strength, but it does not remove the need to consider support, training, or compatibility.

Advantage customisable Advantage often low licence cost Trade-off support may vary
Benefits Potentially lower cost, community development, transparency, and customisation.
Limitations Support quality, training needs, or compatibility may vary between products.
Principle Open source means the code can be inspected or changed, not simply that the software is free of charge.
Decision The best choice depends on the organisation’s skills, needs, and support expectations.

Exam Traps

Common mistakes with software categories and open source

These marks are often lost when answers confuse the user task with the maintenance task, or oversimplify open source.

Mixing up utility and application software

Students sometimes call backup tools "application software" just because a user clicks them.

Fix: Ask whether the software maintains the system or completes the user’s main work.

Assuming open source always means free and perfect

Open source can be strong, but it still has trade-offs.

Fix: Mention support, compatibility, and internal expertise as well as cost.

Ignoring the purpose of the user

A feature-rich package can still be unsuitable if it does not match the task or users well.

Fix: Link the software choice back to the actual job that needs doing.

6 marks Discuss

Question 2

A school is considering open source office software for staff and students. Discuss the potential advantages and disadvantages of that decision.

Show answer guide
  • Open source software may reduce licence costs and allow more flexibility.
  • The school may benefit from transparency or customisation options.
  • However, staff may need training if the interface or features differ from current tools.
  • Compatibility with existing files or workflows must be checked carefully.
  • Support arrangements may vary compared with commercial packages.
  • The final judgement should balance savings against practical implementation and support needs.

Glossary

Quick glossary

Utility software
Software that maintains, protects, or manages the system.
Application software
Software that helps users complete specific tasks.
Open source
Software whose source code is available under an open licence.
Compatibility
How well software works with other systems or files.

Remember

Purpose first

  • Utility software supports the system.
  • Application software supports the user’s main task.
  • Open source can be powerful, but it still needs evaluation.
