Overview

What you should know by the end of this lesson

Quick memory hooks

  • RAID: drive arrangement.
  • NAS: shared network storage.
  • Striping: split data for speed.
  • Mirroring: copy data for resilience.
  • Parity: recovery information.

Key exam warning

RAID can improve fault tolerance, but it does not replace a proper backup strategy. RAID may keep a system available after a drive failure; backups protect against wider data-loss events such as deletion, corruption, ransomware, or site loss.

RAID Mechanisms

RAID levels combine striping, mirroring, and recovery data in different ways

RAID stands for Redundant Array of Independent Disks. The important idea is how data blocks and recovery information are placed across the available drives.

Block-level striping

A1A4A7
A2A5A8
A3A6A9

Data blocks are split across drives. This can increase speed because several drives can work at once.

Mirroring

A1A2A3
A1A2A3

The same data is copied to another drive. If one drive fails, the mirrored copy can still be used.

Parity

A1B1P
A2PC1
PB2C2

Extra parity information is stored so missing data can be rebuilt if a supported drive failure occurs.

Fault tolerance The ability to keep working after a component fails. RAID 0 has no fault tolerance, while mirrored or parity-based RAID can tolerate some drive failures.
Rebuild The process of recreating missing data on a replacement drive using mirrored data or parity information.
Capacity trade-off Some drives or parts of drives are used for copies or parity, so usable capacity is lower than the raw total capacity.
Performance trade-off Striping can improve speed, but parity calculations and rebuilds can add overhead.

RAID Standards

The main RAID levels make different trade-offs

RAID standards describe common ways of arranging drives. For most Unit 2 answers, focus on the main practical levels and know that some older levels exist mainly to explain error-correction ideas.

RAID 0

Striping only

Minimum drives: 2+. Fast, but no fault tolerance.

RAID 1

Mirroring

Minimum drives: 2. Can tolerate one drive failure in a mirror pair.

RAID 5

Block striping with distributed parity

Minimum drives: 3+. Can tolerate one drive failure.

RAID 6

Block striping with double distributed parity

Minimum drives: 4+. Can tolerate two drive failures.

RAID 10

Mirrored pairs plus striping

Minimum drives: 4+. Fast and resilient, but uses more capacity.

Level Main method Minimum drives Fault tolerance Best fit
RAID 0 Block-level striping 2+ None Speed where data loss risk is acceptable or data is temporary.
RAID 1 Mirroring 2 Usually one drive in a mirror pair Simple resilience for important systems.
RAID 5 Block-level striping with distributed parity 3+ One drive Balance of capacity and resilience for shared storage.
RAID 6 Block-level striping with double distributed parity 4+ Two drives Larger arrays where rebuild time and second failures are a risk.
RAID 10 Striping across mirrored pairs 4+ One drive per mirror pair High performance and resilience when capacity cost is acceptable.

What about RAID 2, 3, and 4?

These are less common in modern systems, but they help explain the ideas behind error correction. RAID 2 used bit-level striping with Hamming code error correction. RAID 3 and RAID 4 used parity on a dedicated parity disk, with RAID 3 working at byte level and RAID 4 working at block level.

Error Correction

RAID can use extra information to detect or rebuild missing data

Error correction in RAID is about storing enough extra information to recover from certain failures. The method depends on the RAID level.

Hamming code error correction

D1D2D3H1H2

RAID 2 used Hamming code error correction with data striped at bit level. It needed extra check disks and is now mostly a historical example rather than a common practical choice.

Parity error correction

ABPCP

Parity stores calculated recovery information. If one disk in a supported parity array fails, the missing blocks can be reconstructed from the remaining data and parity.

Method Used by Purpose Exam point
Hamming code RAID 2 Detect and correct errors using check bits. Rare/legacy, but useful to know as an error-correction approach.
Dedicated parity RAID 3 and RAID 4 Store parity on a specific parity disk. The parity disk can become a bottleneck.
Distributed parity RAID 5 Spread parity across drives. Needs 3+ drives and tolerates one drive failure.
Double distributed parity RAID 6 Store enough parity for two drive failures. Needs 4+ drives and gives stronger fault tolerance.

NAS

NAS provides shared storage to users and devices over a network

NAS stands for Network Attached Storage. It is a storage device or appliance connected to a network so users can access shared files without storing everything on their own machines.

Student PC
Teacher laptop
NAS

Shared files, permissions, and storage management

Media workstation
Central storage Files can be stored in one place instead of being scattered across many separate computers.
Shared access Several users or departments can work from the same file space, depending on permissions.
Management Administrators can manage storage, user access, quotas, and sometimes backups from one system.
Network dependence If the network is down or congested, access to NAS files may be affected.

RAID and NAS can appear together

A NAS device may use RAID internally. NAS describes how storage is shared across the network, while RAID describes how the drives inside the storage system are arranged.

Choice

Storage decisions should consider availability, recovery, access, and backups

The best solution depends on how much downtime is acceptable, how many users need access, how important fast recovery is after a failure, and what kind of data-loss risks must be covered.

Need speed only?

RAID 0 can improve performance, but it has no fault tolerance.

Need drive failure protection?

RAID 1, 5, 6, or 10 can keep data available after supported failures.

Need shared access?

NAS gives network users a central storage location.

Need wider recovery?

Backups are still needed for deletion, corruption, attack, or disaster.

Worked judgement

A small design studio may use a NAS with RAID 5 or RAID 6 so staff can share project files and keep working if a drive fails. However, the studio still needs a separate backup plan because RAID will not protect against accidental deletion, ransomware, or the NAS being physically damaged.

Exam Traps

Common mistakes with RAID and NAS

Most confusion comes from mixing up storage sharing, disk arrangement, and backup.

Thinking RAID means backup

RAID can protect against some drive failures, but not every data-loss situation.

Fix: Say that backup is still needed for deletion, corruption, ransomware, and site loss.

Treating NAS as a RAID level

NAS is not a RAID type. It is a network storage approach.

Fix: Separate shared network access from the drive arrangement inside the storage system.

Claiming RAID 0 is resilient

RAID 0 stripes data for speed but has no redundancy.

Fix: State that one disk failure can break the whole RAID 0 array.

Ignoring rebuild risk

Parity arrays may be vulnerable or slower during rebuild.

Fix: Mention downtime, rebuild time, and whether one or two drive failures can be tolerated.

Question 1

Explain how RAID 1 and RAID 5 use different methods to improve fault tolerance.

Show answer guide
  • RAID 1 uses mirroring.
  • The same data is stored on another drive.

  • RAID 5 uses block-level striping with distributed parity.

6 marks Discuss

Question 2

A small design studio wants central file access for staff and reduced disruption if a drive fails. Discuss whether NAS, RAID, or a combination of both would be most suitable.

Show answer guide
  • NAS would help because staff need shared central file access across the network.
  • RAID would help because the studio wants reduced disruption if a drive fails.
  • A NAS device using RAID internally may meet both needs.
  • RAID 5 or RAID 6 could balance capacity and fault tolerance, depending on the number of drives and risk.
  • However, backups are still needed because RAID is not a full backup solution.
  • The strongest judgement is usually that a combination with separate backups is most suitable.
8 marks Analyse

Question 3

A school is choosing between RAID 0, RAID 1, RAID 5, and RAID 6 for a shared file system. Analyse which RAID level would be most suitable and justify your answer.

Show answer guide
  • RAID 0 should not be chosen for important shared files because it has no fault tolerance.
  • RAID 1 is simple and resilient but uses half the raw capacity for mirrored copies.
  • RAID 5 needs at least three drives and can tolerate one drive failure using distributed parity.
  • RAID 6 needs at least four drives and can tolerate two drive failures using double distributed parity.
  • RAID 5 or RAID 6 is likely more suitable for shared file storage than RAID 0 because availability matters.
  • RAID 6 may be better for larger arrays where rebuild time and a second drive failure are realistic risks.
  • Backups are still needed because RAID does not protect against all data-loss events.

Glossary

Quick glossary

RAID
A way of arranging multiple disks to improve resilience, speed, or both.
NAS
Network attached storage that provides shared file access.
Striping
Splitting data blocks across multiple drives.
Mirroring
Keeping duplicate copies of data on more than one drive.
Parity
Recovery information used to rebuild missing data after supported failures.
Fault tolerance
The ability to keep working after a component fails.
Hamming code
Error-correction code associated with RAID 2.

Remember

Separate the ideas

  • RAID is about disk arrangement.
  • NAS is about network-based shared storage.
  • RAID 0 improves speed but has no fault tolerance.
  • Parity can rebuild missing data after supported failures.
  • RAID and NAS do not remove the need for backups.
