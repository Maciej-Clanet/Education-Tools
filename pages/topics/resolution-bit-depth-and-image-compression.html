Overview

What you should know by the end of this lesson

Image quality is not determined by one feature alone. Resolution, bit depth, and compression each affect how an image looks, how much data it needs, and how suitable it is for a task.

Learning goals

  • Explain how resolution affects image detail.
  • Explain how bit depth affects colour range.
  • Compare lossy and lossless compression.
  • Choose suitable settings for different uses.

Quick memory hooks

  • Resolution: how many pixels.
  • Bit depth: how much colour per pixel.
  • Compression: reduce stored data.
  • Trade-off: quality versus size.

Resolution

Resolution affects how much image detail can be stored

Resolution describes the number of pixels used to build the image. More pixels usually mean more detail, but they also increase the amount of data stored.

More pixels more detail Fewer pixels less detail Trade-off higher storage use

Why resolution matters

A small web thumbnail can use lower resolution than a printed poster because it is displayed at a smaller size. The larger the final display area, the more detail may be needed.

Bit Depth

Bit depth controls how many possible colours each pixel can store

Bit depth is the number of bits used for one pixel's colour information. Increasing bit depth allows more colours or shades to be represented, which can improve realism and smooth gradients.

Low bit depth Fewer possible colours, smaller storage requirements.
High bit depth More colours and smoother transitions, but more data needed.
Visual effect Greater colour range can reduce banding in detailed images.
Storage effect Each pixel takes more bits, so the whole file grows.

Compression

Compression reduces file size by removing or reorganising stored data

Compression makes files smaller, which helps storage and transfer. The important distinction is whether the original image can be recovered exactly.

Method What it does Main implication
Lossless Reduces size without losing original image data Image can be restored exactly
Lossy Removes some data to shrink the file further Quality may decrease

Choosing the right compression

If perfect image fidelity matters, lossless compression is safer. If a much smaller file is more important than exact preservation, lossy compression may be chosen.

Trade-Offs

Image choices should match the purpose of the file

Need Likely choice Reason
Small web image Lower resolution or stronger compression Faster loading and smaller storage use
Detailed print work Higher resolution and suitable bit depth Better visual quality
Exact archive Lossless compression Preserves the original data

Exam Traps

Common mistakes with image quality and compression

Mixing up resolution and bit depth

Resolution is about pixel count. Bit depth is about colour per pixel.

Fix: Ask whether the question is about image detail or colour range.

Assuming all compression is lossless

Some compression methods permanently remove data.

Fix: State whether the original image can be recovered exactly.

Ignoring the trade-off

Better quality usually needs more data unless some quality is sacrificed.

Fix: Link image quality decisions to file size and intended use.

4 marks Explain

Question 1

Explain how both resolution and bit depth can increase the storage needed for one bitmap image.

Show answer guide
  • Higher resolution means more pixels must be stored.
  • Higher bit depth means each pixel uses more bits.
  • Both changes increase the total amount of image data.
6 marks Evaluate

Question 2

Evaluate whether lossy or lossless compression is more suitable for storing an important logo archive and a set of quickly loading website images.

Show answer guide
  • Lossless compression preserves the original data exactly.
  • That is useful for an important archive where quality must be preserved.
  • Lossy compression can make website images smaller and faster to load.
  • The trade-off is that some image quality may be lost.

Glossary

Quick glossary

Resolution
The number of pixels used to store the image.
Bit depth
The number of bits used to store each pixel's colour.
Lossless compression
Compression that preserves the original data exactly.
Lossy compression
Compression that removes some data to shrink the file.

Remember

Three linked decisions

  • Resolution controls image detail.
  • Bit depth controls colour precision.
  • Compression controls how size is reduced and at what cost.
